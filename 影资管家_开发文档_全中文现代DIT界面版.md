# 1. 项目概述

## 1.1 项目背景

影视项目中的素材通常包含大量视频、音频、图片、工程文件、文档、字幕和其他辅助文件。素材目录结构复杂，文件体积大，手动整理、检查、交付和归档效率低。

本软件用于帮助 DIT、剪辑师、数据管理人员、制片和归档人员快速完成以下工作：

1. 导入一个或多个顶级素材目录。
2. 自动递归识别下层目录结构。
3. 自动识别视频、音频、图片、文档和其他文件。
4. 对视频自动生成缩略图。
5. 对音频生成封面、波形或默认音频预览图。
6. 对视频/音频提取元数据。
7. 在软件内浏览目录树、素材卡片和技术详情。
8. 导出完整路径文件树形结构图。
9. 导出只包含视频/音频文件的专业元数据 PDF 报表。

# 3. 设计原则

## 3.1 原始素材安全原则

软件只建立本地索引，不修改原始素材。

禁止默认执行以下操作：

1. 移动原始文件。
2. 删除原始文件。
3. 重命名原始文件。
4. 写入原始素材目录。
5. 修改视频、音频、图片的内部元数据。

所有缩略图、缓存、数据库和报表文件均存储在软件数据目录或用户指定导出目录。

## 3.2 本地优先原则

影视素材通常位于本地硬盘、移动硬盘、RAID、NAS 或映射盘。软件应优先采用本地桌面端设计，避免强制上传云端。

## 3.3 异步任务原则

目录扫描、元数据读取、缩略图生成、哈希计算、PDF 导出都可能耗时，必须进入后台任务队列，不允许阻塞 UI。

## 3.4 可恢复原则

对于大项目，扫描和处理过程中断后，应能继续处理未完成任务，而不是全部重来。

## 3.5 报表专业化原则

PDF 报表不是普通文件列表，而是面向影视交付和归档的 DIT 风格技术报告，应包含封面、摘要、缩略图索引、视频明细、音频明细、异常页和文件树附录。

# 5. 功能范围

## 5.1 导入功能

### 5.1.1 支持导入方式

1. 拖拽目录到软件主窗口。
2. 拖拽文件到软件主窗口。
3. 点击「添加目录」按钮。
4. 点击「添加文件」按钮。
5. 菜单栏「文件 > 导入目录」。
6. 菜单栏「文件 > 导入文件」。

### 5.1.2 导入目录规则

导入目录时，软件应：

1. 将该目录作为顶级根目录。
2. 自动递归扫描所有下级目录。
3. 保留原始层级结构。
4. 记录绝对路径和相对路径。
5. 记录文件大小、扩展名、修改时间。
6. 自动识别文件类型。
7. 自动创建缩略图任务和元数据任务。
8. 对不可访问文件记录错误，不中断整个扫描。

### 5.1.3 导入文件规则

导入单个或多个文件时，软件应：

1. 将文件加入当前项目。
2. 以文件所在目录作为来源路径。
3. 记录文件基本信息。
4. 如果是视频/音频，则创建元数据任务。
5. 如果是视频/图片/音频，则创建缩略图或预览任务。

```text

```

## 6.3 图片文件

默认识别扩展名：

```text
.jpg, .jpeg, .png, .tif, .tiff, .bmp, .gif, .webp, .exr, .dpx
```

## 6.4 字幕文件

默认识别扩展名：

```text
.srt, .ass, .ssa, .vtt, .stl, .scc
```

默认识别扩展名：


## 6.6 文档文件

默认识别扩展名：

```text
.pdf, .doc, .docx, .xls, .xlsx, .csv, .txt, .md
```

# 7. 技术选型

## 7.1 推荐技术路线

V1.0 建议采用以下技术栈：

| 模块 | 推荐方案 |
|---|---|
| 客户端框架 | Qt / PySide6 |
| 开发语言 | Python 或 C++ |
| 本地数据库 | SQLite |
| 视频缩略图 | FFmpeg |
| 媒体元数据 | ffprobe / MediaInfo |
| PDF 报表 | 报表Lab / Qt PDF / WeasyPrint |
| 图片处理 | Pillow / Qt Image |
| 音频波形 | ffmpeg + Python 音频解析库 |
| 配置文件 | JSON / YAML |
| 日志 | 本地 log 文件 |
| 打包 | PyInstaller / Nuitka / CMake |

## 7.2 MVP 推荐实现

如果追求快速开发，建议：

```text
Python 3.x
PySide6
SQLite
FFmpeg / ffprobe
报表Lab
Pillow
```

如果追求长期商业化和高性能，建议：

```text
C++ Qt
SQLite
FFmpeg C API 或外部 ffmpeg/ffprobe
Qt PDF / 自研 PDF 渲染模块
```

V1.0 可先采用 Python + PySide6 快速验证产品逻辑，后续如性能瓶颈明显，再将扫描、媒体解析、PDF 导出等模块拆成独立服务或迁移到 C++。

# 9. UI 布局设计：全中文现代 DIT 艺术工作台版

> 本节替换旧版“三栏文件管理器”式布局。新版 UI 不再以传统文件浏览器为中心，而是参考现代 DIT / Media Management 软件的工作流：**导入 导入、素材库 素材库、检查/质检 检查、报表工作室、任务监控**。界面应具有电影工业感、信息密度高但不杂乱、状态反馈清晰、缩略图视觉突出。

---

## 9.1 设计目标

新版 UI 目标：

1. 弱化传统“左树 + 中表 + 右详情”的老式办公软件观感。
2. 强化影视行业视觉语言：缩略图、卡片、胶片条、卷/卡/源盘、检查状态、报表预览。
3. 将“目录结构”升级为“素材源 / 卷 / 卡 / 项目集合”的概念。
4. 将“导出报表”升级为独立的 报表工作室，而不是一个普通弹窗。
5. 将任务状态做成可视化 任务时间线，而不是底部文字状态栏。
6. 提供深色专业主题作为默认主题，浅色主题用于打印和制片交付。
7. 支持多显示器场景：主屏管理，副屏播放/表格/QC。

## 9.3 顶部指挥台 顶部控制台

顶部指挥台 是整个软件的主导航和状态中枢，不做传统密集按钮栏。

## 9.4 Workspace 工作区模型

## 9.13 交互动效

动效必须克制，不做花哨动画。

推荐动效：

1. 拖拽进入窗口时，中央 导入 Drop Zone 发光。
2. 扫描时 素材源卡片 出现环形进度。
3. 缩略图生成后淡入。
4. 任务完成时 任务时间线 出现绿色完成状态。
5. 异常出现时 素材源卡片 边缘轻微红色闪烁一次。
6. 报表工作室 切换模板时预览平滑更新。

禁止：

1. 大面积弹跳动画。
2. 影响性能的模糊动画。
3. 频繁闪烁。
4. 卡片过度 3D 化。
5. 报表预览使用复杂转场。

# 13. 缩略图设计

## 13.1 视频缩略图策略

默认策略：

1. 优先抽取视频时长 10% 位置的一帧。
2. 如果视频时长小于 10 秒，取 1 秒处。
3. 如果该帧为黑帧，尝试 30% 和 50% 位置。
4. 如果仍失败，取首个可解码关键帧。
5. 生成两种尺寸：
   - 小图：320px 宽，用于网格/表格。
   - 大图：960px 宽，用于详情面板和 PDF。

高级选项：

1. 单缩略图。
2. 三帧缩略图：开头、中段、结尾。
3. 缩略图联系表：每个 clip 多帧缩略图。
4. 手动重新生成缩略图。
5. 手动指定缩略图时间点。

## 13.2 音频缩略图策略

音频文件处理顺序：

1. 优先读取内嵌封面。
2. 无封面则生成波形图。
3. 波形生成失败则显示音频格式图标。

## 13.3 缩略图缓存路径

建议结构：

```text
应用数据目录/
└── 影资管家/
    └── cache/
        └── projects/
            └── {project_id}/
                ├── thumbnails/
                │   ├── small/
                │   ├── large/
                │   └── contact_sheet/
                └── waveform/
```

# 20. 性能设计

## 20.1 目录扫描优化

1. 使用后台线程。
2. 分批写入数据库。
3. 每扫描一定数量文件刷新一次 UI。
4. 对网络盘降低并发。
5. 支持取消。
6. 支持中断后重新扫描。

## 20.2 缩略图生成优化

1. 当前可见文件优先生成。
2. 每个任务设置超时时间。
3. 失败任务最多重试 2 次。
4. 缓存命中时不重复生成。
5. 大项目限制并发数量。
6. 缩略图统一压缩为 JPG 或 WebP。

## 20.3 元数据读取优化

1. 只处理视频/音频。
2. 保存 raw metadata，避免重复读取。
3. 文件大小或修改时间变化后才重新读取。
4. 损坏文件不阻塞整体任务。
5. 长时间无响应任务自动失败。

## 20.4 数据库优化

需要建立索引：

```sql
CREATE INDEX idx_asset_project ON asset_file(project_id);
CREATE INDEX idx_asset_root ON asset_file(root_id);
CREATE INDEX idx_asset_folder ON asset_file(folder_id);
CREATE INDEX idx_asset_type ON asset_file(file_type);
CREATE INDEX idx_asset_name ON asset_file(file_name);
CREATE INDEX idx_asset_relative_path ON asset_file(relative_path);
CREATE INDEX idx_metadata_asset ON media_metadata(asset_id);
CREATE INDEX idx_task_status ON task_queue(status);
CREATE INDEX idx_task_type ON task_queue(task_type);
```

# 29. 开发注意事项

1. 不要在 UI 主线程执行扫描、缩略图和元数据任务。
2. 不要默认写入原始素材目录。
3. 不要因为单个损坏文件导致整个任务失败。
4. 不要在 PDF 表格中横向堆积过多字段。
5. 不要把非视频/音频文件混入元数据报表明细。
6. 大项目必须考虑分页、缓存和任务恢复。
7. 路径中可能包含中文、空格、特殊符号和超长路径。
8. 网络盘可能随时断开，需要容错。
9. PDF 报表必须保留导出时间和软件版本。
10. 所有导出结果必须可追溯到项目数据库中的索引数据。

# 30. 总结

本软件 V1.0 的开发核心是：

1. 稳定导入目录。
2. 准确构建文件树。
3. 可靠提取视频/音频元数据。
4. 自动生成缩略图。
5. 导出专业 PDF 报表。
6. 保证原始素材安全。

V1.0 不追求复杂协作功能，应优先保证本地处理的稳定性、报表质量和大目录性能。完成这些基础能力后，再逐步扩展哈希校验、重复检测、PDF 模板编辑、团队协作和归档工作流。

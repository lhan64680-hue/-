# 影资管家官网项目开发规划

## 1. 任务目标

在当前仓库内新增一个与桌面端源码隔离的 `tools-web` 官网项目，最终可部署到用户自己的 Linux 服务器，不依赖公开云托管平台。

官网需要同时完成四件事：

1. 清楚介绍“影资管家”的定位、核心能力、适用场景与使用流程。
2. 提供一个与当前 Qt/QML 桌面端在视觉和操作路径上高度一致的在线 Demo。
3. 提供最新版 Windows 安装包下载入口，并允许服务器直链或 GitHub Release 作为可配置下载源。
4. 迁移并适配 Pond5 官网已有的赞助二维码、赞助申报、赞助榜和管理员确认流程。

## 2. Demo 可行性与实现边界

### 2.1 可以实现的 1:1 范围

- 使用桌面端 `Main.qml` 的 1600×980 设计尺寸作为唯一基准画布。
- 同步 QML 中的主题色、顶部栏 64px 高度、左右栏宽度、边距、圆角、字体层级和控件状态。
- 复现项目库、素材库、素材管理中心、报表、任务、反馈与设置等关键页面的导航和主要操作路径。
- 使用浏览器内模拟数据展示项目打开、素材筛选、卡片选中、预览、批量解析、任务进度、主题切换等效果。
- 在窄屏浏览器中保持原桌面布局比例，通过等比缩放与全屏入口避免组件重新排版造成认知差异。

### 2.2 不能直接 1:1 搬到浏览器的能力

- Windows 原生文件夹选择与完整磁盘扫描。
- SQLite 项目库、FFmpeg/Qt Multimedia 播放链路与本地模型推理。
- 对用户真实素材的删除、移动、批量解析、报表导出和自动更新。
- 原生窗口行为、系统级拖放和与安装目录相关的操作。

上述能力在 Demo 中只做可逆的状态模拟，并始终提示“演示数据，不访问本机文件”。如果未来要求网页真正处理本地素材，应单独立项评估 Qt for WebAssembly、浏览器 File System Access API、服务端转码与隐私合规，不与本期官网混在一起。

## 3. 技术路线

- 官网前端：沿用已验证的原生 HTML、CSS、JavaScript 静态站点结构，便于 Nginx 直接托管，减少服务器依赖。
- 在线 Demo：独立静态子应用，固定 1600×980 设计画布并按容器等比缩放；使用模块化 JavaScript 状态驱动页面。
- 赞助服务：FastAPI + SQLite，沿用 Pond5 站点的“用户申报 → 管理员核实到账 → 公开赞助榜”流程。
- 下载服务：使用 `downloads/release.json` 管理版本、文件大小、发布日期与下载地址；既支持服务器本地安装包，也支持外部备用地址。
- 部署：Nginx 提供静态文件与大文件下载，反向代理赞助 API；systemd/PM2 二选一，以 systemd 示例为主。
- 数据安全：Demo 不上传素材；赞助接口限制字段、金额、频率和来源；管理员操作使用独立令牌且只通过请求头传递。

## 4. 模块拆分

### 模块 0：规划与基线

- 读取最新进度快照、QML 主界面、主题和关键工作区。
- 盘点 Pond5 官网的下载、赞助前端、FastAPI/SQLite 后端和部署配置。
- 固化 Demo 边界、目录结构、迁移映射和验收标准。

### 模块 1：官网基础

- 创建独立 `tools-web` 目录。
- 完成首页导航、首屏、产品定位、功能矩阵、工作流、Demo 入口、下载入口、赞助入口和页脚。
- 接入现有 `logo/logo.png` 与基础 SEO/社交分享信息。
- 完成桌面、平板、手机端响应式样式与无障碍基础。

### 模块 2：高保真在线 Demo

- 建立 1600×980 固定画布与等比缩放器。
- 实现顶部命令栏、项目库、素材源栏、素材卡片区、检查器、任务底栏和设置弹窗。
- 复现项目打开、标签页切换、搜索筛选、素材选择、预览、批量解析、任务进度与主题切换。
- 对所有原生或危险能力使用模拟反馈，不触碰用户本地文件。

### 模块 3：赞助功能

- 迁移微信、支付宝赞助二维码和前端交互。
- 迁移赞助申报、申报状态查询、公开赞助榜和管理员审核页。
- 将 Pond5 命名空间全部改为 CineVault/影资管家。
- 增加输入校验、蜜罐、频率限制、管理员令牌校验和 SQLite 测试。

### 模块 4：下载与自托管部署

- 增加最新版本配置与安装包元数据展示。
- 默认接入当前正式版 `v0.1.146`，支持后续只改一个 JSON 即更新官网。
- 提供服务器本地安装包路径与 GitHub Release 备用地址。
- 提供 Nginx、systemd、环境变量示例和一键同步脚本，不执行公开云部署。

### 模块 5：全局验证

- 静态资源、链接、API 和赞助状态测试。
- Python 后端单元测试与启动烟测。
- Demo 关键交互回归与 1600×980 布局基线检查。
- 检查移动端、键盘操作、减少动画偏好和无脚本降级。
- 清理无用临时缓存并确认未影响桌面端现有未提交改动。

### 模块 6：快照与交付

- 生成新的递增进度快照。
- 给出本地预览、服务器部署、版本更新和赞助审核说明。

## 8. 验收标准

1. 官网可由 Nginx 直接托管，后端只需 Python/FastAPI/SQLite，不依赖公开云平台。
2. 首页在 360px、768px、1440px 宽度下无横向溢出，主要按钮和导航可用。
3. Demo 在 1600×980 基线下与 QML 的主尺寸常量一致，并在其他尺寸下只等比缩放、不擅自重排桌面软件布局。
4. Demo 至少覆盖项目库、素材库、素材管理中心、任务、设置五条关键体验路径。
5. Demo 不读取、上传或修改访客本地文件，所有模拟行为有清晰提示。
6. 下载按钮展示版本、发布日期、文件大小和来源；主地址失败时保留备用地址；同一窗口明确展示自愿赞助和联系方式，且不阻断免费下载。
7. 赞助申报不会未经管理员确认直接公开；管理员令牌错误时返回 401；重复提交受频率限制。
8. 后端测试全部通过，关键页面和 API 完成烟测。
9. 不修改或覆盖当前桌面端自然语言搜索模块的未提交代码。
10. `dit.ee2x.cn` 使用独立 TLS 证书、Nginx 配置、systemd 服务和 SQLite 数据库，部署与更新不影响 `pd.ee2x.cn`。

# 影资管家全盘索引、无感解析与素材中心优化实施方案

> 日期：2026-07-15  
> 基线：当前工作区 v0.1.164 未提交开发状态  
> 实施原则：本机优先、增量更新、可恢复、低前台干扰、兼容旧项目库

## 一、任务目标

1. 支持把固定磁盘、移动磁盘或任意目录作为素材源，一次加入后递归索引全部可访问文件。
2. 递归监测素材源变化；系统连续无输入 30 分钟后，自动执行增量扫描、真实元数据提取、文档正文提取与待解析内容补全。
3. 所有“打开所在目录”入口统一为打开 Windows 资源管理器并选中目标文件。
4. 文档自动提取正文并生成摘要，正文、摘要和关键词进入全局搜索。
5. 集成 ExifTool，读取视频、音频、图片的真实嵌入元数据；保存原始 JSON，并把常用字段归一化后写入数据库和搜索索引。
6. 素材管理中心所有素材卡片显示稳定、连续、可辨识的结果序号。
7. 取消解析结果人工确认：历史待确认记录自动迁移为已完成，成功解析结果自动生效，移除确认按钮、筛选项、状态标识和提示文案。
8. 修复现有扫描、项目切换、后台任务生命周期和搜索竞态等已知问题，并降低大库扫描、数据库同步和 QML 渲染开销。

## 二、关键产品约束

### 1. “全盘解析”分为两层

- 本地基础解析：文件名、路径、类型、大小、时间、ExifTool 元数据、文档正文。此层默认全量执行，不依赖模型。
- 模型增强解析：图片视觉摘要、视频逐帧解析、文档摘要。此层在系统空闲 30 分钟后分批执行；用户恢复输入后停止派发新任务，已进入安全提交阶段的单个任务允许完成。

这样既能保证所有文件可搜索，也不会在用户工作时抢占 CPU、GPU、磁盘和网络资源。

### 2. 不把全盘扫描建立在逐目录 QFileSystemWatcher 上

Windows 下使用递归目录变化通知，只把素材源标记为“脏”；空闲维护时再做增量核对。应用启动、监测溢出、磁盘重连后执行低频兜底核对，避免漏变更。

### 3. 旧数据库保持可迁移

确认状态字段先保留为兼容字段，但不再暴露给 UI；迁移时把历史待确认结果转为已完成，后续成功解析自动写入已完成状态。确认链路稳定后再考虑物理删除旧字段。

### 4. 扫描不得先删除旧索引

新扫描写入独立 generation/暂存数据，完整成功后原子切换；失败、取消、崩溃或磁盘拔出时继续保留上一份完整目录和分析结果。

## 三、模块拆分与实施顺序

### 模块 0：数据一致性与后台任务基座

- 扫描引入 generation/暂存表或等价的原子切换机制。
- 后台任务携带不可变项目 UUID、数据库路径、缓存根目录和任务代次。
- 项目切换和程序退出建立任务屏障，禁止旧项目任务写入新项目数据库。
- SQLite 主连接和线程连接统一启用外键、WAL、busy timeout 与同步级别。

### 模块 1：统一文件定位

- 新增共享 `FileRevealService`。
- Windows 优先调用系统 Shell 定位 API；失败时回退 `explorer.exe /select`，最后才回退到打开父目录。
- 替换素材库、素材管理中心、快捷搜索和详情区域的所有旧入口。

### 模块 2：素材中心交互收口

- 卡片序号使用当前结果模型的 1 基索引；视觉帧卡片同时保留帧号，避免混淆。
- 移除确认筛选、按钮、角标、状态文本和批量确认入口。
- 成功解析直接自动生效；历史待确认数据迁移后立即参与搜索。

### 模块 3：磁盘卷与全盘索引

- 新增 `StorageVolumeService`，枚举就绪卷、卷标、文件系统、容量、可用空间、是否只读/可移除。
- “添加素材源”增加“添加磁盘卷”入口；同一路径规范化去重，禁止把项目数据库缓存目录递归加入自身。
- 扫描器容忍无权限目录、断开的重解析点、循环链接、超长路径和临时消失文件。
- 记录跳过目录/文件数量和原因，但不因个别文件失败终止整卷扫描。

### 模块 4：ExifTool 真实元数据

- 新增 `ExifToolAdapter` 与 `MetadataExtractionService`。
- 读取单文件 JSON，保存原始结果、工具版本、提取时间和文件指纹。
- 归一化字段至少包括：拍摄时间、创建时间、厂商、机型、镜头、序列号脱敏值、GPS 可选值、方向、尺寸、时长、帧率、编解码器、色彩空间、采样率、声道、码率。
- 元数据摘要进入全局 FTS 和语义文档；文件未变化时按指纹跳过重复提取。
- 构建与安装器携带 ExifTool 运行时，并在缺失时给出明确降级状态。

### 模块 5：文件变化监测与空闲调度

- 新增递归 `SourceChangeMonitor`，按素材源合并短时间高频事件。
- 新增 `SystemIdleMonitor`，Windows 使用系统最后输入时间，不把“用户切到其他软件”误判为电脑空闲。
- 新增 `BackgroundMaintenanceCoordinator`：变化标脏 → 空闲 30 分钟 → 增量扫描 → 元数据/正文 → 全局目录同步 → 模型补充解析 → 搜索索引同步。
- 用户恢复输入后停止派发；低电量、只读卷、离线卷、数据库忙或前台已有重任务时自动延后。

### 模块 6：文档正文与摘要闭环

- 扩展 DOCX/XLSX/PPTX、TXT/MD/CSV/TSV/JSON/SRT/ASS/VTT 正文提取。
- PDF 使用可用的本地文本提取器；旧 DOC/XLS/PPT 无可靠解析器时明确标记仅元数据索引，不伪造摘要。
- 正文以长度和哈希去重，摘要失败不清空已提取正文；摘要任务可重试、可续跑。
- 文档正文、摘要、关键词和元数据共同进入搜索。

### 模块 7：全方位搜索与性能优化

- 搜索覆盖项目、素材源、目录、文件名、路径、扩展名、类型、真实元数据、文档正文、AI 摘要、OCR、视觉帧和时间范围。
- 为常用过滤字段建立普通索引，为全文字段使用 FTS5；避免 `%LIKE%` 扫描大表。
- 元数据、正文和语义索引按文件指纹增量更新，删除项使用批量事务处理。
- UI 使用分页/虚拟化、结果 generation 防竞态、异步查询和节流刷新。

### 模块 8：已知 Bug、安全与发布收口

- 修复审计报告中的 P0/P1：反馈链路明文与默认密码、更新包验签、跨项目异步串库、任务退出悬空、破坏性项目操作、附件鉴权与流式上传、SQLite 连接一致性。
- 增加扫描故障注入、项目切换并发、磁盘拔出、监测溢出、空闲恢复、元数据异常、旧库迁移和大库性能测试。
- 完成 Release 构建、安装包、升级安装、真实磁盘烟测和缓存清理。

## 六、验收标准

1. 可直接选择 `C:\`、移动盘根目录或普通目录作为素材源；无权限目录不会导致整次扫描失败。
2. 在任意深度新增、修改、重命名、移动或删除文件后，系统能标记变化；空闲 30 分钟后自动更新目录、元数据和搜索结果。
3. 用户恢复鼠标/键盘输入后，不再继续派发后台解析；前台搜索、预览和滚动无明显卡顿。
4. 扫描中断、磁盘拔出、数据库忙、程序退出后，上一次完整索引仍可搜索，重启可恢复未完成任务。
5. 所有“打开所在目录”入口都打开资源管理器并选中目标文件；目录结果打开目标目录本身。
6. 文档正文与摘要可搜索；摘要失败时正文仍保留，文件修改后会自动重新提取。
7. 视频、音频、图片能查看 ExifTool 原始元数据和归一化摘要；未修改文件不会重复提取。
8. 素材卡片显示正确序号；界面不再出现待确认、已确认、确认按钮或确认筛选。
9. 旧项目库自动升级且数据不丢失；全量 CTest、迁移测试、故障注入测试、QML 检查和 Release 构建通过。
10. 10 万文件级素材源的增量变更不触发全量模型重跑；内存、CPU、磁盘队列和界面响应满足性能基线。

## 七、建议性能基线

- 前台空闲时 UI 主线程单次同步工作不超过 16ms，重查询不阻塞主线程。
- 10 万文件首次扫描峰值内存控制在 500MiB 以内，数据库批次提交可配置。
- 单文件变更到“已标记待维护”不超过 5 秒；进入空闲后 60 秒内开始维护。
- 文件未变化时，增量核对不得重新调用 ExifTool、正文提取或模型解析。
- 后台任务默认低优先级，模型增强任务并发数默认 1，元数据提取并发按磁盘类型限流。

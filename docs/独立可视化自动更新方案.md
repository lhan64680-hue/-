# 影资管家独立可视化自动更新方案

## 目标

影资管家在 Windows 上采用与 `storyboard` 项目一致的更新体验：主程序负责检查 GitHub Release 和下载安装包，安装阶段交给从当前部署目录复制出的独立更新器运行时。独立更新器等待旧进程退出、静默执行 Inno Setup、展示安装进度，并在成功后自动启动新版本。

## 用户流程

1. 主程序启动后后台检查 GitHub Latest Release，或用户在设置页手动点击“检查更新”。
2. 发现新版本后下载到应用数据目录的 `cache/updates/windows`，先写 `.part`，完成后校验 GitHub 资产大小。
3. 自动更新关闭时，用户可选择“立即更新”或“下次启动更新”；自动更新开启时直接进入安装流程。
4. 主程序把当前部署运行时复制到 `cache/updates/windows/staging/{session}_runtime`，跳过业务数据和 FFmpeg 工具目录。
5. 临时 `CineVault.exe` 使用更新会话参数启动独立进度窗口，主程序在确认启动成功后退出。
6. 独立更新器依次展示“准备安装、关闭旧版本、安装新版本、启动新版本、完成”。
7. 更新器通过 PowerShell 提权运行 Inno Setup 静默安装，成功后启动安装目录中的新版 `CineVault.exe`。
8. 更新失败时窗口保留错误信息，安装包和会话日志不删除，便于重试和排查。

## 模块职责

| 模块 | 文件 | 职责 |
| --- | --- | --- |
| Release 与下载 | `src/application/UpdateService.*` | 版本检查、代理、下载、缓存、更新策略、启动独立更新器 |
| 更新会话 | `src/application/UpdaterSession.*` | 会话参数、临时运行时、等待旧进程、静默安装、日志、新版重启 |
| 更新窗口 | `src/ui/widgets/UpdaterWindow.*` | 五阶段进度、成功/失败状态、运行期间阻止误关闭 |
| 启动入口 | `src/app/main.cpp` | 区分正常主程序和独立更新会话，更新模式不初始化完整业务 |
| 更新设置 | `AppSettings.*`、`SettingsViewModel.*`、`SettingsDialog.qml` | 自动更新、立即更新、下次启动更新和下载代理设置 |
| 安装器 | `installer/windows/cinevault.iss` | 静默安装兼容、安装目录覆盖、静默时不重复启动主程序 |

## 更新会话参数

```text
--run-update-session={sessionId}
--update-version={versionTag}
--update-installer={installerPath}
--update-install-root={installRoot}
--update-executable-name=CineVault.exe
--update-old-pid={pid}
```

只有参数完整且版本号有效时才进入更新器模式。无更新参数时继续正常启动；存在会话标志但参数损坏时显示明确错误，不加载数据库和完整 QML 界面。

## 静默安装参数与日志

PowerShell 使用 `Start-Process -Verb RunAs -Wait -PassThru` 执行安装包，并传入：

```text
/SP-
/VERYSILENT
/SUPPRESSMSGBOXES
/NORESTART
/NOCANCEL
/CLOSEAPPLICATIONS
/FORCECLOSEAPPLICATIONS
/DIR="{当前安装目录}"
/LOG="{会话目录}/installer.log"
```

每次会话日志位于：

```text
cache/updates/windows/sessions/{sessionId}/installer.log
cache/updates/windows/sessions/{sessionId}/updater.log
```

Inno `[Run]` 保留 `skipifsilent`，因此静默更新时由独立更新器统一启动新版，避免安装器和更新器重复拉起程序。

## 状态策略

- `autoInstallUpdates=false` 为默认值，下载完成后询问用户。
- 选择“下次启动更新”会持久化对应版本；下次正常启动时若安装包仍有效，会自动打开独立更新器。
- 自动更新开启后，下载或复用有效更新包时直接安装。
- 待安装包必须存在、文件名符合版本规则且在复用 Release 资产时大小一致。
- 待安装版本不高于当前版本、文件丢失或命名无效时，清理 pending 与下次启动标志。
- 下载新版本会清理旧版本的下次启动标志，避免误装旧包。

## 发布与验收

Windows 安装包名称继续使用：

```text
CineVault-Setup-v{version}.exe
```

发布前至少执行：

```powershell
ctest --test-dir dit-tools-src\cinevault-pro\build\windows-msvc-release-real --output-on-failure
.\tool\build_windows.ps1 -RealWorkflow
```

端到端验收应从已安装旧版本开始，确认：检测并下载新版、立即更新窗口出现、旧版退出、管理员确认、静默安装、新版自动启动、设置页版本更新，以及失败场景日志可用。

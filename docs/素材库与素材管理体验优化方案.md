## 实现决策

- 空格全屏播放只作用于已选中的视频素材；非视频素材只保留选择能力。
- 缩略图尺寸和源素材折叠状态先做会话级 QML 状态，不写入设置或数据库。
- 旧视频摘要不自动重写；更详细摘要在重新解析或新解析后生效。
- 所有 QML 新增调用必须同步补齐真实 ViewModel 与 Minimal ViewModel，避免最小 GUI 构建断裂。
- 顶部切换卡顿优先从 QML 工作区生命周期修复：减少 `Loader` 切换造成的组件销毁/重建，并控制素材管理中心进入页面时的 reload 频率。
- 安装器不再依赖 Inno/Restart Manager 的默认“关闭应用”提示；安装初始化、正式写文件前和卸载初始化都主动执行 `taskkill /F /T /IM CineVault.exe`，并最多重试 3 次。
- 若用户正常向导安装，`NextButtonClick(wpReady)` 会先执行同一套进程清理，再进入 Inno 自身的准备安装/文件占用检测阶段，避免旧进程在默认检测前仍存活。
- 播放器通过“已有视频帧”状态控制缩略图海报可见性：未播放时显示缩略图，播放、暂停、拖动或逐帧后保持视频输出画面。
- 素材库左右键保持双用途：视频正在播放或已有预览帧时交给播放器逐帧；其他状态继续用于素材选择，避免破坏既有键盘浏览。
- 单帧步长优先读取 QtMultimedia 的视频帧率元数据；读取不到时回退到 40ms。

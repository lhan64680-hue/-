# 反馈发送失败修复-前后对比

## 阶段信息

- 备份版本：`backup/v0.1.117`
- 阶段目标：修复桌面端“使用反馈”页无法发送反馈信息的问题。

## 根因

- 反馈输入框 `draftMessageArea` 位于 `chatWorkspaceComponent` 的 `Loader` 组件树内部。
- 上一轮把 `submitDraftMessage()` 与 `onMessageSubmitted()` 写在了 `root` 级别，直接引用了 `draftMessageArea`。
- 当 `Loader` 尚未创建出聊天工作区，或 `root` 级作用域无法访问该局部 id 时，会报：

```text
ReferenceError: draftMessageArea is not defined
```

- 这个异常会打断发送与发送后清空输入框的逻辑，因此表现为“软件端无法发送反馈信息”。

## 修改文件

- `dit-tools-src/cinevault-pro/src/ui/qml/workspaces/FeedbackWorkspace.qml`

## 关键对比

修改前：

```qml
function submitDraftMessage() {
    if (viewModel) {
        viewModel.sendMessage(draftMessageArea.text)
    }
}

Connections {
    target: viewModel
    function onMessageSubmitted(success) {
        if (success) {
            draftMessageArea.text = ""
        }
    }
}

onClicked: root.submitDraftMessage()
```

修改后：

```qml
function submitDraftMessage(text) {
    if (viewModel) {
        viewModel.sendMessage(text)
    }
}

Connections {
    target: viewModel
    function onMessageSubmitted(success) {
        if (success) {
            draftMessageArea.text = ""
        }
    }
}

onClicked: root.submitDraftMessage(draftMessageArea.text)
```

调整说明：

- `root` 层不再直接读取 `draftMessageArea`
- 将输入文本显式作为参数传给 `submitDraftMessage(text)`
- 清空输入框的 `Connections` 下沉回 `chatWorkspaceComponent` 内部，保证与 `draftMessageArea` 处于同一作用域

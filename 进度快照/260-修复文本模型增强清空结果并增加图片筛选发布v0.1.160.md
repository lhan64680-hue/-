## 根因分析

### 1. 模型属性被直接升级为硬条件

`SearchQueryUnderstanding::merge` 原先会把模型返回实体中的 `colors / materials / attributes` 直接合并进 `strictEntities`。这些字段随后由查询服务按“同一帧必须全部满足”执行硬过滤。

0.6B 文本模型即使正确识别了“男人”和“牛仔裤”两个实体，也可能补出用户没有说过的颜色、材质或“穿着”等关系属性。这些推断字段在视觉索引中不存在时，第二轮增强搜索就会把首轮结果全部排除。

### 2. 真实索引全部是旧版非结构化帧事实

排查本机数据库 `material-center.sqlite`：

- 素材：228 条。
- 视觉帧：14,238 条。
- `facts_complete=1`：0 条。
- `facts_complete=0`：14,238 条。

库中实际存在大量“男人/男子 + 牛仔裤”的同帧结果，所以不是素材缺失。旧版帧依靠描述、标签和对象文本回退匹配，而原实现要求实体标签精确包含“男人”，不会把“男子、男性、男士”视为同义词，进一步放大了零结果问题。

## 已完成内容

### 1. 模型硬条件原文落地

- 新增 `groundedModelProperties` 和 `groundedModelEntity`。
- 模型提供的颜色、材质、属性只有明确出现在用户原句中，才能成为硬数据库条件。
- “穿着、穿戴、戴着、拿着、手持、正在、坐着、站着、走着”等关系或动作词不会作为实体硬属性。
- 用户明确说出的实体仍然保留，例如“男人”和“牛仔裤”继续要求同帧共现，不会退回到“所有牛仔裤都命中”。
- 更新本地模型系统提示，要求模型不得依据常识补充原句没有的性质。

### 2. 旧版帧实体同义匹配

- 查询服务增加实体标签别名组：
  - 男人 / 男性 / 男子 / 男士
  - 女人 / 女性 / 女子 / 女士
  - 牛仔裤 / 长裤 / 裤子 / 丹宁裤
- 结构化事实和旧版同帧文本回退统一使用别名匹配。
- 颜色、材质、属性仍要求同帧文本明确出现，避免扩大误命中。

### 3. 图片快捷筛选

- `SearchResultQuickFilter` 增加 `Image`。
- 快捷筛选顺序统一为：智能、视频、帧画面、图片、文档。
- 图片筛选强制结果目标为素材，并设置 `AssetType::Image`。
- 点击图片按钮时同步顶部素材类型为图片；手动修改类型下拉框时仍回到智能筛选。
- 主界面和快捷搜索框共享同一筛选状态和查询重跑逻辑。

## 关键代码前后对比

### 模型实体属性

修改前：模型属性直接成为硬条件。

```cpp
changed = mergeEntity(&merged.strictEntities, entity) || changed;
```

修改后：先以用户原句为事实边界。

```cpp
changed = mergeEntity(
              &merged.strictEntities,
              groundedModelEntity(entity, localQuery.originalText))
    || changed;
```

### 旧版帧实体匹配

修改前：所有字段都做精确文本包含。

```cpp
for (const auto &term : constraint.allTerms()) {
    if (!searchable.contains(term.simplified().toCaseFolded())) {
        return false;
    }
}
```

修改后：实体标签支持可靠同义词，属性仍严格匹配。

```cpp
const auto labelAliases = entityLabelAliases(constraint.label);
const bool labelMatched = std::any_of(
    labelAliases.cbegin(), labelAliases.cend(), [&searchable](const QString &alias) {
        return searchable.contains(alias.simplified().toCaseFolded());
    });
```

### 图片快捷筛选

```cpp
case SearchResultQuickFilter::Image:
    parsed->resultTarget = SearchResultTarget::Assets;
    parsed->assetTypeFilters = {static_cast<int>(AssetType::Image)};
    parsed->assetTypeFilter = static_cast<int>(AssetType::Image);
    break;
```

# 215-旧帧状态回填修复

## 已完成内容

- 已定位截图报错原因：旧基础帧已有解析内容，但 `analysis_state` 仍是 `Pending(0)`，新逐帧多维度逻辑只认 `Success(1)`，因此误判为没有成功帧。
- 已新增 schema 7 迁移：
  - Pending + 有基础解析内容 + 无错误 -> 回填为 Success。
  - Pending + 有错误信息 -> 回填为 Failed。
  - Pending + 无内容 + 无错误 -> 保持 Pending。
- 已给 `successfulFrames(...)` 增加运行时兼容兜底：旧库里 Pending 但已有有效基础描述且无错误的帧，也视为可用于逐帧多维度解析。
- 已新增迁移测试，覆盖旧帧状态回填。
- 已打包并安装 `v0.1.137`。
- 已启动新版触发实际数据库迁移，并确认安装目录数据库已修复旧帧状态。

## 当前修改到哪个模块

当前模块：素材管理中心 / 视频基础帧状态迁移 / 多维度逐帧解析前置筛选。

涉及重点：

- `VideoAnalysisService`：成功帧筛选兼容旧 Pending 帧。

## 具体修改的代码前后对比

### 数据库迁移

修改前：schema 6 只保证字段和表存在，没有回填旧帧状态。

```cpp
if (version < 6) {
    setSchemaVersion(db, 6, errorMessage);
}
```

修改后：schema 7 自动修复旧数据。

```cpp
if (version < 7) {
    UPDATE video_frame_analysis SET analysis_state = 1
    WHERE analysis_state = 0
      AND error_message 为空
      AND 已有基础解析内容;

    UPDATE video_frame_analysis SET analysis_state = 2
    WHERE analysis_state = 0
      AND error_message 非空;

    setSchemaVersion(db, 7, errorMessage);
}
```

### 成功帧筛选

修改前：

```cpp
if (frame.analysisState == FrameAnalysisState::Success && frame.errorMessage.trimmed().isEmpty()) {
    items.append(frame);
}
```

修改后：

```cpp
const auto isLegacySuccessfulFrame = frame.analysisState == FrameAnalysisState::Pending
    && hasAnalysisContent
    && frame.errorMessage.trimmed().isEmpty();

if ((frame.analysisState == FrameAnalysisState::Success || isLegacySuccessfulFrame)
    && frame.errorMessage.trimmed().isEmpty()) {
    items.append(frame);
}
```

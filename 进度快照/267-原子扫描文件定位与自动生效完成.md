# 267 - 原子扫描、文件定位与自动生效完成

## 本轮目标

完成总体方案的模块 0、模块 1、模块 2：先消除自动扫描的数据一致性风险，再统一素材文件定位交互，并移除素材管理中心人工确认流程、补齐卡片序号。

## 已完成内容

### 1. 原子扫描与项目上下文隔离

- 项目数据库版本升级到 4，`asset_file` 新增并回填 `path_key`，为大小写不敏感的稳定文件身份提供索引。
- `ScanEngine` 改为先写 TEMP 暂存表，完整扫描成功后才在单个事务中对生产表做更新、插入、删除和目录替换。
- 扫描中断、权限错误或测试注入失败时，旧目录和素材目录保持不变；重试成功才原子切换。
- 未变化文件保留素材 ID、收藏状态、媒体元数据与缩略图；只让新增或实际变化的文件重新进入媒体任务。
- 扫描、媒体任务均捕获入队时的项目数据库路径，避免切换项目后后台线程误写当前项目。
- SQLite 工作线程统一启用 WAL、外键、busy timeout 等连接配置。
- 增加扫描故障注入测试，覆盖“失败保留旧目录、成功重试、素材身份/收藏保持”。

### 2. 打开目录后选中文件

- 新增 `src/shared/FileRevealService.h/.cpp`。
- Windows 素材定位统一调用 `explorer.exe /select,<文件路径>`，直接打开资源管理器并选中素材。
- 非 Windows 或系统不支持选中时安全回退为打开父目录。
- 素材库、精简素材库、素材管理中心的素材定位入口全部接入统一服务；纯目录结果仍直接打开该目录。

### 3. 素材管理中心卡片序号

- 素材卡片和帧卡片缩略图左上角新增明显的 `#序号` 徽标。
- 保留原列表排序序号，使快速扫描和搜索命中定位一致。
- UI 契约测试新增 `frameRankText`、`assetRankText` 覆盖。

### 4. 移除人工确认流程

- 移除素材管理中心确认筛选、全部确认按钮、单素材确认按钮、确认状态标签及对应 ViewModel/QML 接口。
- 移除 `VideoAnalysisService::confirmVideo/confirmVideos` 人工确认入口。
- 图片、文档、视频分析成功后直接显示“结果已自动生效”。
- 全局数据库升级到版本 12：历史 Ready 记录自动迁移为已生效，并回填 `confirmed_at`。
- 兼容字段暂时保留在数据库和查询底层，避免旧数据库/搜索结构破坏，但不再向用户暴露，也不参与素材中心筛选。

## 具体修改前后对比

### 扫描提交

修改前：

```text
扫描开始 -> 删除旧 asset_file/folder_node -> 分批写新数据 -> 扫描失败会留下半成品
```

修改后：

```text
扫描开始 -> TEMP 暂存全量结果 -> 校验完成 -> BEGIN IMMEDIATE
        -> 保留身份地合并素材 -> 替换目录 -> 更新统计 -> COMMIT
扫描失败 -> 丢弃 TEMP，旧目录完全不动
```

### 文件定位

修改前：

```cpp
QDesktopServices::openUrl(QUrl::fromLocalFile(info.absolutePath()));
```

修改后：

```cpp
FileRevealService::revealFile(asset.absolutePath, &errorMessage);
// Windows: explorer.exe /select,<目标文件>
```

### 解析状态

修改前：

```text
解析成功 -> 等待确认 -> 用户点击确认 -> 结果生效
```

修改后：

```text
解析成功 -> 自动写入生效状态 -> 立即进入搜索与展示
```

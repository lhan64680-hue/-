# 进度快照 247：语义索引多核非阻塞优化与本机部署完成

## 已完成内容

### 代码优化

- BGE 从逐条单线程式推理改为批量 ONNX 推理，默认批大小 32。
- ONNX Runtime 默认使用 `逻辑核心数 - 2` 个 intra-op 线程；本机 28 线程 CPU 默认配置为 26，保留 2 个逻辑核心维持 UI 和系统调度。
- 支持通过 `CINEVAULT_BGE_THREADS` 和 `CINEVAULT_BGE_BATCH_SIZE` 调优。
- 语义索引生成阶段不再长期持有搜索互斥锁。
- 更新期间继续提供旧索引，完成后短暂原子换入新索引。
- 搜索入口使用 try-lock，不在 UI 路径同步触发重建或无限等待。
- 后台索引进度已接入 ViewModel 和素材管理中心 QML；完成后自动刷新当前搜索。

关键前后对比：

```text
原实现：逐文档 ONNX Run + 搜索/更新共用长锁 -> 约 1 核工作、搜索等待、窗口假死
新实现：批量 ONNX Run + 26 推理线程 + 锁外计算 + 旧索引服务 -> 约 25 核工作、窗口可响应
```

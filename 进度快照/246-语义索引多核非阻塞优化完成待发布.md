# 进度快照 246：语义索引多核非阻塞优化完成，待全量发布

## 任务目标

解决“素材管理中心首次搜索看似卡死”的问题，并将语义索引建立从单核串行执行优化为可配置的多核批量推理，同时保证索引更新期间界面可响应、旧索引可搜索、进度可见。

## 已完成内容

### 1. 根因定位

- 安装目录：`D:\Program Files\影资管家`。
- Windows 事件记录确认原版本出现 `AppHangB1`。
- 实际原因是启动后建立全量语义索引；旧实现按帧单条生成向量，并且搜索与索引更新共用同一把全局互斥锁。
- 用户搜索时 UI 线程等待长任务释放锁，因此表现为卡死；旧进程 CPU 利用率约等效 1 核。

### 2. 多核批量推理

修改模块：

- `src/core/search/BgeEmbeddingModel.h`
- `src/core/search/BgeEmbeddingModel.cpp`

关键前后对比：

```text
优化前：每个文档单独 tokenize -> 单条 ONNX Run -> 单条归一化
优化后：文档按批次 tokenize/padding -> [batch, sequence] ONNX Run -> 批量提取与归一化
```

```text
优化前：ONNX Runtime 使用默认线程行为，没有应用级可调参数
优化后：intra-op 默认使用 logical CPU - 2，inter-op 固定 1；批大小默认 32
```

可调环境变量：

- `CINEVAULT_BGE_THREADS`：推理线程数，范围 1-256。
- `CINEVAULT_BGE_BATCH_SIZE`：批大小，范围 1-64。

### 3. 非阻塞搜索与稳定旧索引

修改模块：

- `src/core/search/SemanticSearchIndexService.h`
- `src/core/search/SemanticSearchIndexService.cpp`

关键前后对比：

```text
优化前：搜索、建模、写索引全程共用一把全局锁；搜索可能等待数分钟
优化后：耗时向量生成不持有搜索锁；旧索引继续服务；最终提交阶段才短暂换入新索引
```

```text
优化前：搜索入口可能同步触发 ensure/rebuild
优化后：搜索只读取已就绪索引，try_lock 失败立即回退，不在 UI 路径等待或建索引
```

### 4. 后台进度和完成后自动刷新

修改模块：

- `src/application/SearchDocumentSyncService.h/.cpp`
- `src/ui/viewmodels/MaterialCenterViewModel.h/.cpp`
- `src/ui/qml/workspaces/MaterialCenterWorkspace.qml`
- `src/app/AppContext.cpp`

新增能力：

- 后台同步发布收集、批量向量生成、提交三个阶段的进度。
- ViewModel 暴露 `semanticIndexing`、`semanticIndexProgress`、`semanticIndexStatusText`。
- 素材管理中心显示进度条和百分比，并明确搜索不会等待后台更新。
- 索引完成后自动刷新当前搜索结果。

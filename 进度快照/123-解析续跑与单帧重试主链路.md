# 123-解析续跑与单帧重试主链路

日期：2026-06-30

## 已完成内容

- 已创建第二阶段备份：`backup/v0.1.105`。
- 已将 `VideoAnalysisService` 队列从 `QQueue<QString>` 升级为 `QQueue<AnalysisJob>`。
- 已补齐 4 种任务模式：
  - `Initial`
  - `Resume`
  - `Rebuild`
  - `SingleFrame`
- 已实现真正的续跑逻辑：
  - 有已落库帧结果时不再先删库。
  - 汇总失败后可直接从汇总阶段继续。
  - 只有显式重建才清空旧结果和缓存。
- 已实现单帧自动重试：
  - 单帧失败自动最多重试 3 次
  - 第 3 次失败后标记 `Skipped`
  - 整条视频继续处理，不再全盘回滚
- 已实现单帧手动补解析入口：
  - `VideoAnalysisService::retryFrame(videoKey, frameNumber, ...)`
- 已让汇总阶段支持降载重试：
  - 全量帧 + 900 tokens
  - 抽样 18 帧 + 700 tokens
  - 抽样 12 帧 + 550 tokens
- 已让视觉接口非 200 错误尽量带回正文 message，并写日志。

## 当前修改到哪个模块

当前完成模块：解析续跑与单帧重试主链路模块。

核心解析路径已经从“每次删库重跑”切换成“按任务模式续跑 / 重建 / 单帧补跑”，服务层主逻辑已具备断点恢复能力。

## 具体修改的代码前后对比

### 修改前

```text
- enqueueVideo 只传 videoKey。
- startNextAnalysis 一开始就 deleteAnalysisArtifacts + removeDirectoryContents。
- 所有失败帧都只累加 failedFrames，不能续跑也不能单独补解析。
- summarizeVideo 只发一次完整汇总请求，失败直接整条任务 Failed。
```

### 修改后

```text
- 解析队列改为 AnalysisJob，支持 Initial / Resume / Rebuild / SingleFrame。
- Resume 模式优先读取 video_frame_analysis 和 video_analysis_task 续跑。
- 单帧自动 3 次重试，第三次失败转为 Skipped，不阻塞全局。
- Summarize 支持抽样降载重试，并把重试次数落到 task 表。
- 非 200 会尽量提取服务端 message，并写日志。
```

# 273 - 快捷搜索双击主窗口拉起生命周期修复完成

## 本轮目标

修复快捷搜索结果双击后，工程与目标素材能够切换，但主程序窗口无法恢复显示的问题；覆盖素材、帧画面和文件夹结果，并验证隐藏到托盘后的 Windows 前台激活。

## 根因

- 运行日志在用户双击时间点连续记录：`QuickSearchWindow.qml:704: ReferenceError: root is not defined`。
- 结果 delegate 内先同步调用 `openQuickSearchResultAtIndex()` 或 `openQuickSearchFolderResult()`。
- 上述调用会切换工程、重载搜索模型并销毁当前 delegate；函数返回后再执行 `root.showMainWindow(true)` 时，delegate 的 QML 上下文已经失效。
- 因而真正故障发生在窗口恢复入口之前。旧测试直接调用控制器，只验证了窗口可见，未覆盖 delegate 被同步销毁的生命周期。

## 已完成内容

### 1. 将结果激活流程移到持久根窗口

- `QuickSearchWindow.qml` 新增：
  - `activateFolderResult(folderKeyValue, locateOnly)`；
  - `activatePrimaryResult(videoKeyValue, resultIndex, locateOnly)`。
- 素材/帧和文件夹 delegate 的 `activate()` 只负责把稳定值传给根窗口函数。
- 工程切换导致 delegate 销毁后，根窗口函数仍能继续调用 `showMainWindow(true)`。
- 打开失败时也会恢复主窗口，以便用户看到业务错误信息。

修改前：

```qml
if (root.materialCenterViewModel.openQuickSearchResultAtIndex(videoKeyValue, index)) {
    root.showMainWindow(true)
}
```

修改后：

```qml
function activate(locateOnly) {
    root.activatePrimaryResult(videoKeyValue, index, locateOnly)
}
```

根窗口持久上下文：

```qml
viewModel.openQuickSearchResultAtIndex(videoKeyValue, resultIndex)
showMainWindow(true)
```

### 2. 把 Windows 恢复升级为可验证状态机

- 原生恢复不再以 `IsWindowVisible()` 作为成功标准，只以 `GetForegroundWindow() == windowHandle` 判断真正置前。
- 使用目标窗口实际线程 ID 与当前前台线程执行临时 `AttachThreadInput`。
- 新增 0/50/150/300ms 分阶段重试，覆盖快捷搜索隐藏、Qt 事件提交和 Windows 前台切换的竞态。
- 新增 `isMainWindowForeground()` 与 `mainWindowRestoreFinished(bool)`，可测试真实结果而非仅测试请求是否发出。

修改前：

```cpp
return IsWindowVisible(windowHandle) != FALSE;
```

修改后：

```cpp
return GetForegroundWindow() == windowHandle;
```

### 3. 回归测试增强

- `QuickSearchControllerTest`：
  - 隐藏窗口恢复后验证真实前台 HWND；
  - 增加竞争窗口重新抢焦点场景，验证重试状态机夺回主窗口。
- `MaterialCenterUiContractTest`：
  - 锁定根窗口激活函数；
  - 禁止把会重建模型的调用重新放回 delegate；
  - 锁定严格前台判断与重试数组契约。

# 271 - 快捷搜索托盘双击恢复主窗口完成

## 本轮目标

修复主窗口隐藏到系统托盘或被其他窗口遮挡后，通过全局快捷键打开快捷搜索并双击结果时，主窗口不能稳定恢复到前台的问题；保持既有的自动进入目标工程、切换素材管理中心、滚动并选中素材或目标帧能力不变。

## 已完成内容

### 1. 定位根因

- `QuickSearchWindow.qml` 原流程先隐藏快捷搜索窗口，再请求主窗口 `showNormal/raise/requestActivate`。
- 主窗口已经隐藏到托盘时，快捷搜索窗口是本进程当前持有前台权限的窗口；先隐藏它会丢失 Windows 前台激活链，随后对主窗口的激活请求可能被系统拒绝。
- 主窗口本来就在前台时不会暴露该问题，因此与用户反馈的现象一致。

### 2. 统一主窗口恢复入口

- `Main.qml` 新增 `restoreToForeground()`，统一执行：
  1. `showNormal()`；
  2. `raise()`；
  3. `requestActivate()`。
- 托盘“显示主窗口”和应用内 `Ctrl+K` 均改用该入口，避免窗口恢复逻辑继续分散。

### 3. 修复快捷搜索关闭与激活顺序

- `QuickSearchWindow.qml` 新增 `activateMainWindow()`，优先调用主窗口统一入口，并保留兼容回退逻辑。
- `showMainWindow()` 改为：
  1. 快捷搜索仍持有前台权限时先恢复主窗口；
  2. 再关闭快捷搜索；
  3. 下一事件循环再次激活主窗口，确保快捷搜索隐藏后主窗口成为最终前台窗口。
- 素材、目标帧和文件夹双击仍走原有原子导航链路；`Ctrl+Enter` 仅在资源管理器定位文件的语义未改。

### 4. 增加回归契约

- `MaterialCenterUiContractTest.cpp` 新增窗口恢复顺序检查：
  - 主窗口必须按 `showNormal -> raise -> requestActivate` 执行；
  - 快捷搜索必须按 `先激活主窗口 -> 隐藏快捷搜索 -> 延迟再次激活主窗口` 执行。

## 具体修改前后对比

修改前：

```qml
if (forceCloseQuickSearch || !pinned) {
    hideSearch()
}
mainWindow.showNormal()
mainWindow.raise()
mainWindow.requestActivate()
```

修改后：

```qml
root.activateMainWindow()
if (shouldCloseQuickSearch) {
    hideSearch()
}
Qt.callLater(function() {
    root.activateMainWindow()
})
```

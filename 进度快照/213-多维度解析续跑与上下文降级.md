# 213-多维度解析续跑与上下文降级

- 时间：2026-07-06 09:02:46
- 当前模块：素材管理中心全局多维度解析稳定性

## 已完成内容

- 已读取上一份最新快照：`212-全局多维度解析入口.md`。
- 已创建阶段备份：`backup/v0.1.164`。
- 已分析截图中的失败原因：多维度解析请求将基础解析上下文一次性提交给视觉接口，旧逻辑最多截取 28000 字符，部分模型会返回 `Context size has been exceeded`。
- 已确认多维度解析不是把全部视频帧重新解析一遍：
  - 不重新抽帧。
  - 不重新调用逐帧视觉解析。
  - 只读取数据库中已有基础摘要和成功帧文本，最多采样一部分帧描述作为补充分析上下文。
- 已实现全局多维度解析“继续未完成”：
  - 新增 `VideoAnalysisService::pendingDimensionCount(...)`。
  - 全局入口会跳过所选维度已全部完成的素材。
  - 对部分完成的素材，只提交缺失维度。
  - 如果任务启动后发现维度已存在，按成功完成，不再显示失败。
- 已修复上下文超限：
  - 多维度请求按 12000 / 8000 / 4000 / 2000 字符逐级降级。
  - 遇到 400、413 或 context/token/length/too large 类错误时自动缩短上下文重试。
- 已新增回归测试：接口首次返回 `Context size has been exceeded` 时，客户端会缩短上下文再次请求并成功解析。
- 已生成阶段代码对比文档：
  - `backup/v0.1.164/多维度解析续跑与上下文降级-代码对比.md`

修改前：

```cpp
bool retryFrame(const QString &videoKey, int frameNumber, QString *errorMessage = nullptr);

```

修改后：

```cpp
bool retryFrame(const QString &videoKey, int frameNumber, QString *errorMessage = nullptr);
int pendingDimensionCount(const QString &videoKey, const QStringList &dimensions, QString *errorMessage = nullptr) const;
bool analyzeDimensions(const QString &videoKey, const QStringList &dimensions, QString *errorMessage = nullptr);
```

### 全局入口续跑过滤

修改前：

```cpp
QString errorMessage;
if (m_analysisService->analyzeDimensions(asset.videoKey, dimensionNames, &errorMessage)) {
    ++accepted;
}
```

修改后：

```cpp
const auto pendingCount = m_analysisService->pendingDimensionCount(asset.videoKey, dimensionNames, &pendingError);
if (pendingCount == 0) {
    ++completed;
    continue;
}

QString errorMessage;
if (m_analysisService->analyzeDimensions(asset.videoKey, dimensionNames, &errorMessage)) {
    ++accepted;
}
```

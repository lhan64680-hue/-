# 批量解析补充与全量重建策略完成

## 已完成内容

- 修复“批量解析”在已有旧数据时只能处理待解析/失败素材、无法补齐新版结构化字段的问题。
- 批量解析检测到当前结果存在已完成、失败或中断的旧解析状态时，弹窗提供三种明确选择：
  - `补充解析（推荐）`
  - `全部重新解析`
  - `取消`
- 补充解析按素材状态分流：
  - 待解析素材使用 `Initial`；
  - 失败或中断素材使用 `Resume`；
  - 已完成素材先检查当前结构化解析版本、采样计划及逐帧完整性，仅存在缺口时使用 `Resume`；
  - 已完整素材直接跳过，不进入模型调用队列。
- 续跑路径继续复用 `VisualAnalysisMetadata::isFrameAnalysisComplete`，只请求缺少颜色、材质、属性、品牌/文字/OCR 等当前结构化字段的帧，完整帧不会重复解析。
- “全部重新解析”改为显式分配 `AnalysisRunMode::Rebuild`；修复旧实现虽然按钮写着“全部重新解析”，但 Ready 且存在缺口的素材仍可能自动走 Resume 的语义不一致。
- 新弹窗用推荐主按钮突出补充解析，用危险色按钮标识高成本全量重建，并说明耗时与接口费用差异。
- 新增批量策略 UI/服务合约测试，锁定补充路径不能出现 `Rebuild` 或清理旧数据调用。

### 1. 已有数据的批量入口

修改前：

```qml
ActionButton {
    text: "解析未完成素材"
    onClicked: viewModel.analyzeVisiblePending()
}
```

修改后：

```qml
ActionButton {
    text: "补充解析（推荐）"
    primary: true
    onClicked: viewModel.analyzeVisibleSupplement()
}

ActionButton {
    text: "全部重新解析"
    danger: true
    onClicked: viewModel.analyzeVisibleAll()
}
```

### 2. 已完成素材的补充判定

修改前：

```cpp
// analyzeVisiblePending 只收集 Pending / Failed / Running，Ready 素材不会进入队列。
```

修改后：

```cpp
if (asset.analysisStatus == VideoAnalysisStatus::Ready) {
    const auto hasVisualGap = hasIncompleteVisualFrames(db, normalizedKey, &completenessError);
    if (!hasVisualGap) {
        ++alreadyComplete;
        continue;
    }
    job.mode = AnalysisRunMode::Resume;
}
```

### 3. 全部重新解析强制策略

修改前：

```cpp
m_analysisService->enqueueVideos(videoKeys, &message);

```

修改后：

```cpp
m_analysisService->enqueueVideosForRebuild(videoKeys, &message);

// 服务层
enqueueJob(AnalysisJob{normalizedKey, AnalysisRunMode::Rebuild, 0}, &rejection);
```

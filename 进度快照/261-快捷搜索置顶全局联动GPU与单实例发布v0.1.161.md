# 261 - 快捷搜索置顶、全局联动、GPU 与单实例发布 v0.1.161

## 任务目标

1. 快捷搜索框增加图钉按钮和说明文字，固定后失去焦点或进入主窗口时不自动消失。
2. 快捷搜索改为跨全部项目搜索，不继承当前项目卡片、来源或状态过滤。
3. 双击快捷搜索结果后进入主窗口，打开素材所属项目并自动选中该素材。
4. 内置 Qwen3 0.6B 文本模型从 CPU 切换为 Vulkan GPU 加速，禁止静默回落 CPU。
5. 增加单实例锁，重复启动时唤醒已有主窗口并退出新进程。
6. 完成全量回归、发布构建、真实双进程探针和缓存清理。

## 当前模块

全部开发、测试和发布工作已经完成，当前版本为 `v0.1.161`。

## 已完成内容

### 1. 快捷搜索置顶

- 快捷搜索标题栏新增 `📌 固定显示 / 已固定` 按钮和动态说明。
- 固定后停止失焦收起计时器；切换到主窗口时快捷搜索继续显示。
- 取消固定后恢复原有失焦自动收起行为。
- 启动探针会定位 `quickSearchPinButton`、切换 `pinned=true` 并验证窗口可见性。

### 2. 全项目搜索与结果联动

- 打开快捷搜索时统一清空项目、来源、素材类型、解析状态和确认状态过滤，再执行一次全项目加载。
- 单击结果只更新当前选择；双击或按 Enter 才打开结果。
- 打开结果时先定位素材所属项目，再把主界面切换到素材中心并重新选择准确的 `videoKey`。
- 主界面通过既有选中索引联动自动滚动到目标卡片。

### 3. GPU 文本模型

- llama.cpp 运行时从 CPU 包切换为官方 Windows x64 Vulkan 包。
- 启动前执行 `--list-devices`，只在检测到 GPU 时启动模型。
- 模型参数从 `--n-gpu-layers 0` 改为 `--n-gpu-layers 99`。
- 无 GPU 时明确停用模型增强，并保留确定性本地搜索，不会让模型改走 CPU。
- 设置页显示实际 GPU 名称；本机验证设备为 `NVIDIA GeForce RTX 4070 Ti SUPER`。

### 4. 单实例锁与窗口唤醒

- 新增独立 `SingleInstanceGuard` 模块，使用用户级 `QLockFile` 阻止多开。
- 主实例同时监听用户级 `QLocalServer`；第二实例连接后发送激活请求并立即退出。
- Windows 第二实例授权已有进程获取前台焦点，主实例执行恢复、置前和激活。
- 主窗口尚在启动时收到激活请求会暂存，QML 加载完成后再执行。
- 更新器会话和分析探针保留在单实例锁之前，避免影响它们作为辅助进程运行。
- 异常退出遗留的陈旧锁会在无法连接旧实例后自动清理。

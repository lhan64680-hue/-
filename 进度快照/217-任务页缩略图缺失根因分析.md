### 任务页显示条件

现有逻辑：任务页只在 `selectedSubjectThumbnailPath` 非空时显示图片；为空时显示占位文案。

```qml
readonly property string selectedThumbnailPath: hasSelectedTask ? viewModel.selectedSubjectThumbnailPath : ""
readonly property string selectedThumbnailSource: selectedThumbnailPath.length > 0 && localImageUrlHelper
    ? localImageUrlHelper.sourceForInput(selectedThumbnailPath)
    : ""

Image {
    source: selectedThumbnailSource
    visible: selectedThumbnailSource.length > 0
}
```

定位结论：截图出现“该任务暂无可显示缩略图”，说明该任务当前 `subject.thumbnailPath` 为空，而不是图片路径存在但加载失败。

### 任务对象缩略图来源

现有逻辑：解析任务创建时先只记录素材 key 和名称，后台线程读取 `global_video_asset` 后才补齐路径和缩略图。

```cpp
subject.thumbnailPath = asset.assetType == AssetType::Image ? asset.absolutePath : asset.thumbnailPath;
```

定位结论：视频素材的任务缩略图依赖 `global_video_asset.thumbnail_path`。

### 全局索引同步条件

现有逻辑：全局索引从项目库同步缩略图时，只把成功状态的缩略图路径写入全局库；非成功状态会写空路径。

```sql
CASE WHEN COALESCE(th.status, 0) = 1 THEN COALESCE(th.image_path, '') ELSE '' END
```

定位结论：如果项目库里的缩略图状态不是 `Success = 1`，任务页拿到的 `thumbnailPath` 就会是空。

## 根因判断

主因不是任务页图片控件渲染失败，而是缩略图生成任务曾经中途停止，导致目标素材没有生成缩略图文件，数据库状态永久卡在 `Running` 且路径为空。

更深层设计问题：

- `MediaTaskService::markThumbnailsRunning()` 会先把整批素材全部写成 `Running`。
- 后续再逐个生成并写成 `Success` 或 `Failed`。
- 如果程序中途关闭、崩溃、线程停止，尚未处理到的素材会一直保持 `Running + 空 image_path`。
- 当前代码没有在启动或打开项目时恢复这些陈旧 Running 缩略图，也没有自动重跑。

补充观察：

- QML 日志里有 WebP 不支持警告，但发生在 `MaterialCenterWorkspace.qml:523`，不是任务详情页的 `JobProgressInspectorPane.qml`。
- 目标素材是 mp4，当前问题是 `thumbnailPath` 为空，不是 WebP 路径加载失败。

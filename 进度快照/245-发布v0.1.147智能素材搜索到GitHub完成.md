## 三、发布前发现并修复的问题

### 问题

原 `tool/build_windows.ps1` 只把 `CineVault.exe` 复制到安装包暂存目录，再运行 `windeployqt`。Qt 部署工具不会复制项目自己的 `onnxruntime.dll` 和 `data/models/bge-small-zh-v1.5`，因此第一次生成的候选安装包缺少本地语义搜索运行时资产。

第一次候选包：

- 大小：167,745,705 字节。
- SHA-256：`523EC22CF4E2896AF6163F35D2E4C1ED14B81AB66A09BCF0AB1683D571D5F460`。
- 未发布，随后已安全删除并重建。

### 修改前

```powershell
Copy-Item $exePath -Destination $stagingDir -Force
$deployedExe = Join-Path $stagingDir "CineVault.exe"
Invoke-VcVarsCommand "windeployqt ..."
```

### 修改后

```powershell
$hasOnnxRuntime = Test-Path $onnxRuntimeSource
$hasLocalSearchData = Test-Path $localSearchDataSource
if ($hasOnnxRuntime -xor $hasLocalSearchData) {
    throw "Local-search runtime assets are incomplete ..."
}
if ($hasOnnxRuntime -and $hasLocalSearchData) {
    Copy-Item $onnxRuntimeSource -Destination $stagingDir
    Copy-Item $localSearchDataSource -Destination $stagingDir -Recurse
}
```

效果：

- ONNX Runtime 与模型目录必须成对存在。
- 任一资产缺失时打包立即失败，禁止生成降级或无法启动的安装包。
- 最终安装包体积增至 187,092,974 字节，Inno Setup 压缩输入包含新增资产。

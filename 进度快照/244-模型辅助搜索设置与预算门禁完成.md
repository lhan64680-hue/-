## 四、本轮关键修改与前后对比

### 1. MaterialCenterViewModel 门禁顺序

修改文件：

- `dit-tools-src/cinevault-pro/src/ui/viewmodels/MaterialCenterViewModel.cpp`

修改前：

```cpp
if (模型配置不完整) {
    return;
}
if (缓存命中) {
    return 缓存结果;
}
if (!tryConsumeSearchModelCall(date)) {
    return 预算耗尽;
}
```

问题：模型配置检查早于缓存和预算状态，既不符合约定优先级，也会在 API Key 临时缺失时阻止已有缓存继续使用。

修改后：

```cpp
if (缓存命中) {
    return 缓存结果;
}
if (!canUseSearchModel(date)) {
    return 预算耗尽并保留本地结果;
}
if (模型配置不完整) {
    return 配置状态并保留本地结果;
}
if (!tryConsumeSearchModelCall(date)) {
    return 预算耗尽并保留本地结果;
}
发起真实网络请求;
```

查询理解与候选帧复核均使用同一顺序：

`仅本地搜索` ＞ `对应功能开关` ＞ `缩略图授权（帧复核）` ＞ `缓存/在途复用` ＞ `预算可用性` ＞ `模型配置` ＞ `请求前扣减`。

效果：

- 缓存命中不调用 `tryConsumeSearchModelCall()`。
- 配置缺失不会误扣预算。
- 预算耗尽时优先显示预算状态。
- 网络请求一旦即将发起便先计数，因此网络失败仍计数。
- 所有拒绝路径均保留本地结果或本地排序。

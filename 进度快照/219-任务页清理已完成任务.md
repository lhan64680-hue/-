## 具体修改的代码前后对比

### 1. JobEngine 清理能力

修改前：只有 `clearJobs()`，会清空全部内存任务，不适合作为任务页按钮使用。

```cpp
void clearJobs();
```

修改后：新增只清理完成任务的方法。

```cpp
void clearJobs();
void clearCompletedJobs();
```

实现要点：

```cpp
for (const auto &job : m_jobs) {
    if (job.state != JobState::Completed) {
        keptJobs.append(job);
    }
}
```

如果项目库打开，会同步 best-effort 删除 `job` 表里 `state = Completed` 的记录；随后发出 `jobsChanged()` 刷新页面。

### 2. ViewModel 暴露按钮状态和调用入口

修改前：QML 没有任务清理入口。

修改后：

```cpp
Q_PROPERTY(bool canClearCompletedJobs READ canClearCompletedJobs NOTIFY stateChanged)
Q_INVOKABLE void clearCompletedJobs();
```

`canClearCompletedJobs()` 只在当前任务列表存在 `Completed` 时返回 true。

### 3. QML 顶部按钮

修改前：任务页顶部只有标题。

```qml
Text {
    text: "任务"
}
```

修改后：标题右侧增加按钮。

```qml
ActionButton {
    text: "清理已完成"
    enabled: viewModel && viewModel.canClearCompletedJobs
    onClicked: if (viewModel) viewModel.clearCompletedJobs()
}
```

### 4. 测试

新增 `JobEngineTest::clearCompletedJobsKeepsActiveAndFailedJobs()`：

```cpp
engine.completeJob(completedId, QStringLiteral("完成"));
engine.failJob(failedId, QStringLiteral("失败"));
engine.clearCompletedJobs();

QVERIFY(!containsJob(jobs, completedId));
QVERIFY(containsJob(jobs, runningId));
QVERIFY(containsJob(jobs, pendingId));
QVERIFY(containsJob(jobs, failedId));
```

# 240 - 视觉模型辅助自然语言搜索增强完成

## 一、本轮目标

实现“更懂用户”的自然语言素材检索，重点覆盖：

- `昨天拍摄的视频、图片`：按执行搜索时的本机日期计算昨天，只返回视频或图片。
- `昨天的文件夹`：只返回目录，不混入素材文件。
- `昨天拍摄的视频和图片所在的文件夹`：先以同一素材实体执行日期/类型约束，再返回所属目录。
- `雨夜里有人撑伞`：用已解析的视觉帧描述与本地语义向量检索，即使文件名不包含这些词也能命中。
- `画面文字写着“新品发布”`：要求真实帧 OCR 证据，不能因路径或摘要偶然包含文字而误命中。

### 模块 1：查询理解与结果域

涉及文件：

- `dit-tools-src/cinevault-pro/src/domain/SearchTypes.h`
- `dit-tools-src/cinevault-pro/src/core/search/NaturalLanguageQueryParser.h`
- `dit-tools-src/cinevault-pro/src/core/search/NaturalLanguageQueryParser.cpp`

完成项：

- 新增日期范围、日期字段偏好、结果目标、类型并集、文件夹跨实体意图、OCR 意图和查询解释标签。
- 相对日期以 `QDate::currentDate()` 为默认参考时间，支持今天、昨天、前天、大前天、N 天前、最近 N 天、本周、上周、本月、上月和明确日期。
- `拍摄、录制、素材、帮我找、找、画面` 等交互词不再污染必须检索的内容词。
- 默认结果域为素材；只有明确出现“文件夹/目录”才返回文件夹。
- `视频和图片` 解析为 OR 并集，不会错误理解成同一个文件同时是两种类型。
- 增加雨夜、夜景、航拍、撑伞等视觉兜底词以及引号 OCR 抽取。

代码行为前后对比：

- 之前：`昨天拍摄的素材` 会把“拍摄”留作关键词，容易产生 0 结果。
- 现在：解析为日期硬条件，语义词为空，并显示 `拍摄日期：YYYY-MM-DD`。
- 之前：搜索结果可能混合目录和素材。
- 现在：`SearchResultTarget` 明确隔离两个结果域。

完成项：

- 全局数据库升级到 schema v11，新增 `capture_time`、`capture_date`、`capture_time_source`、`capture_time_confidence` 及日期索引。
- 拍摄时间优先级：QuickTime creation date → EXIF DateTimeOriginal → media creation_time → 目录日期 → 文件修改时间兜底。
- 同步目录时从原始媒体元数据解析并保存真实来源与置信度。
- 除素材级聚合文档外，为每个已解析帧建立 `frame:<videoKey>:<frameNumber>` 视觉搜索文档。
- 帧文档包含描述、标签、对象、动作、场景、实体、OCR 与时间点，并进入现有 BGE/USearch 增量索引。

代码行为前后对比：

- 之前：日期过滤只依据 `modified_at`，容易把拷贝时间误当拍摄时间。
- 现在：有真实拍摄时间时优先使用；只有兜底时间时保留低置信度并明确标注来源。
- 之前：长视频只有一个聚合搜索文档，局部镜头语义容易被稀释。
- 现在：帧级文档独立召回，并映射回素材及具体时间点。

## 具体修改的代码前后对比

### 1. 项目打开后自动恢复陈旧 Running 缩略图

修改前：项目切换后只触发既有项目/素材刷新链路，没有针对陈旧 Running 缩略图的恢复入口。

修改后：

```cpp
connect(m_projectService,
        &ProjectService::projectChanged,
        m_mediaTaskService,
        &MediaTaskService::recoverStaleThumbnails);
```

`MediaTaskService::recoverStaleThumbnails()` 扫描条件：

```sql
WHERE th.status = Running
  AND COALESCE(th.image_path, '') = ''
  AND af.asset_type IN (Video, Image)
```

命中后为对应素材源创建 `JobType::Thumbnail` 恢复任务，并由后台线程调用 `runStaleThumbnailRecovery()` 重新纳入缩略图生成流程。

### 2. 缩略图 Running 标记改为逐素材处理

修改前：

```cpp
markThumbnailsRunning(db, assets, &errorMessage);
for (int i = 0; i < assets.size(); ++i) {
    const auto &asset = assets.at(i);
    // 生成缩略图
}
```

这会在任务开始时把整批素材全部置为 Running；如果进程中断，尚未处理到的素材会永久停留在 Running 且无路径。

修改后：

```cpp
for (int i = 0; i < assets.size(); ++i) {
    const auto &asset = assets.at(i);
    if (!markThumbnailRunning(db, asset.id, &stateError)) {
        failJob(thumbnailJobId, stateError);
        return;
    }
    // 生成当前素材缩略图
}
```

这样只有真正开始处理的素材会被置为 Running，降低中断后大批素材卡死的概率。

### 3. 任务页统计从真实任务列表回退计算

修改前：`hasBatchSummary()` 和顶部统计主要依赖 `VideoAnalysisService` 的素材解析批次；如果没有批次摘要，即使下方存在任务，顶部也可能显示 `0/0`。

修改后：

```cpp
bool JobTimelineViewModel::hasBatchSummary() const
{
    return hasAnalysisBatchSummary(m_videoAnalysisService) || !m_jobs.isEmpty();
}

int JobTimelineViewModel::batchTotalCount() const
{
    return hasAnalysisBatchSummary(m_videoAnalysisService)
        ? m_videoAnalysisService->batchTotalCount()
        : m_jobs.size();
}
```

其他 `batchFinishedCount()`、`batchSuccessfulCount()`、`batchFailedCount()`、`batchQueuedCount()`、`batchProgress()` 和当前任务文案也增加同样的任务列表回退逻辑。

### 4. 进行中任务置顶

修改前：

```cpp
m_items = items;
```

修改后：

```cpp
m_items = items;
sortForDisplay(m_items);
```

排序优先级：

```cpp
Running -> Pending -> Failed -> Completed -> Cancelled
```

同状态下按 `updatedAt` 新的在前，最后按 `id` 倒序。

### 5. 任务详情缩略图状态更明确

修改前：

```qml
text: hasSelectedTask ? "暂无可显示缩略图" : "点击任务卡片后显示素材或对象详情"
```

修改后：

```qml
text: hasSelectedTask ? viewModel.selectedSubjectThumbnailStatusLabel
                      : "点击任务卡片后显示素材或对象详情"
```

`selectedSubjectThumbnailStatusLabel` 根据 `ThumbnailStatus` 返回“缩略图生成中，完成后会自动显示。”、“缩略图生成失败，可重新生成后查看。”、“缩略图等待生成。”等更准确文案。

### 6. JobSubject 携带缩略图状态

修改前：任务详情主体只有 `thumbnailPath`，无法区分“没有缩略图”和“正在生成/失败/等待生成”。

修改后：

```cpp
ThumbnailStatus thumbnailStatus = ThumbnailStatus::Pending;
```

`VideoAnalysisService::analysisSubjectForAsset()` 会把素材的 `thumbnailStatus` 填入 `JobSubject`，任务详情页据此显示状态。

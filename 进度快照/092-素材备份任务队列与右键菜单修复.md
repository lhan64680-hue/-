# 092-素材备份任务队列与右键菜单修复

日期：2026-06-27

## 已完成内容

- 已在阶段开始前执行源码备份，生成 `backup/v0.1.76`。
- 已实现素材备份等待队列：
  - “添加到备份任务”会校验当前待备份源和备份目的地。
  - 添加成功后任务进入等待队列并保存。
  - 添加成功后自动清空待备份源和备份目的地，方便继续添加新任务。
  - “开始备份”只执行队列，按添加时间顺序逐个处理。
  - 未开始任务可在右侧任务列表移除。
  - 执行中的任务不做断点续传，启动任务时从持久化等待队列移除，避免重启后误重复写入半成品目录。
- 已实现按项目持久化素材备份等待队列：
  - 使用项目数据库路径生成项目级 key。
  - 队列保存源路径、目的地路径、主目标、校验模式、级联开关、批次名、创建时间等信息。
- 已修改“添加磁盘卷”：
  - 不再打开文件夹选择器。
  - 使用 `QStorageInfo::mountedVolumes()` 列出已挂载且可用的磁盘分区。
  - 选择后以 `BackupSourceKind::Volume` 加入待备份源。
- 已修复右键菜单只显示细线问题：
  - `ThemedMenu` / `ThemedMenuItem` 补齐稳定尺寸。
  - 素材库、项目库右键弹出改为绑定触发 MouseArea 的可靠坐标。
- 已重新构建、测试、启动烟测和打包。
- 已新增阶段溯源文档：`backup/v0.1.76/素材备份任务队列与右键菜单修复-前后对比.md`。

## 具体修改的代码前后对比

### 1. AppSettings 队列持久化

修改前：没有素材备份等待队列的持久化接口。

```cpp
// AppSettings 未提供素材备份队列 JSON 的读取和保存。
```

修改后：新增项目级素材备份队列 JSON 读写接口。

```cpp
QString materialBackupQueueJson(const QString &projectDatabasePath) const;
void setMaterialBackupQueueJson(const QString &projectDatabasePath, const QString &json);
```

```cpp
const QByteArray digest = QCryptographicHash::hash(normalized.toUtf8(), QCryptographicHash::Sha1).toHex();
return QStringLiteral("materialBackup/queues/%1").arg(QString::fromLatin1(digest));
```

### 2. MaterialBackupViewModel 队列执行

修改前：点击开始备份直接执行当前页面中的源和目的地。

```cpp
void MaterialBackupViewModel::startBackup()
{
    // 基于当前草稿计划并立即执行。
}
```

修改后：先入队并持久化，开始备份时按 FIFO 顺序逐个执行。

```cpp
Q_INVOKABLE void enqueueBackupTask();
Q_INVOKABLE void removeQueuedTask(const QString &taskId);
```

```cpp
void MaterialBackupViewModel::startNextQueuedTask()
{
    m_activeQueuedTask = m_queue.takeFirst();
    persistQueue();
    // 执行当前队列任务，结束后继续下一个。
}
```

### 3. 添加磁盘卷

修改前：使用目录选择器。

```cpp
QFileDialog::getExistingDirectory(...);
```

修改后：枚举系统已挂载卷。

```cpp
const QList<QStorageInfo> volumes = QStorageInfo::mountedVolumes();
```

### 4. 素材备份页面

修改前：备份目的地区域没有入队按钮。

```qml
ActionButton {
    text: "开始备份"
}
```

修改后：目的地底部增加“添加到备份任务”，右侧等待任务可以移除。

```qml
ActionButton {
    text: "添加到备份任务"
    enabled: viewModel && viewModel.canAddBackupTask
    onClicked: viewModel.enqueueBackupTask()
}
```

```qml
ActionButton {
    text: "移除"
    visible: model.status === "Pending" && viewModel && !viewModel.running
    onClicked: viewModel.removeQueuedTask(model.destinationId)
}
```

### 5. 右键菜单

修改前：菜单宽高不稳定，弹出坐标直接使用裸 `mouse.x` / `mouse.y`。

```qml
cardContextMenu.popup(mouse.x, mouse.y)
```

修改后：菜单组件声明稳定尺寸，并使用触发 MouseArea 映射坐标。

```qml
implicitWidth: Math.max(184, contentItem ? contentItem.implicitWidth + leftPadding + rightPadding : 184)
implicitHeight: Math.max(12, contentItem ? contentItem.implicitHeight + topPadding + bottomPadding : 12)
```

```qml
cardContextMenu.popup(cardMouseArea, mouse.x, mouse.y)
rowContextMenu.popup(rowMouseArea, mouse.x, mouse.y)
projectContextMenu.popup(projectMouseArea, mouse.x, mouse.y)
```

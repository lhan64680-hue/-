# 236 - 官网统一下载赞助弹窗与联系方式完成

## 本轮目标

将影资管家官网的“立即下载”体验与参考项目统一：下载和自愿赞助放在同一个组合弹窗中，并迁移开发者联系方式与 QQ 群入口，让用户下载时能够明确知道赞助功能，同时保证不赞助也可免费下载全部功能。

2. 弹窗首屏保留版本、平台、文件大小、发布日期、下载源状态、发布说明和“下载最新版”按钮。
3. 明确加入“软件始终可以免费下载”“不赞助不会影响下载，也不会限制任何功能”的说明。
4. 在同一窗口展示微信和支付宝赞助二维码，支持点击/键盘放大预览。

## 关键代码前后对比

### 下载与赞助结构

修改前：

```html
<div data-download-modal>仅显示安装包信息和下载按钮</div>
<div data-support-modal>单独显示赞助二维码</div>
```

修改后：

```html
<div data-download-modal>
  <section class="download-file-card">版本化下载入口</section>
  <p class="support-message">免费下载与自愿支持说明</p>
  <div class="support-modal-layout">
    <section class="support-payment-panel">二维码与赞助申报</section>
    <aside class="support-contact-panel">联系方式与 QQ 群</aside>
  </div>
</div>
```

### 赞助提交逻辑

修改前：首页和赞助页如果都支持提交，需要分别维护 API、LocalStorage 和请求代码。

修改后：

```javascript
window.CineVaultSupport = Object.freeze({
  api,
  readPending,
  savePending,
  submitClaim,
});
```

`script.js` 与 `sponsors.js` 均调用 `CineVaultSupport.submitClaim()`，避免出现两套行为不一致的赞助逻辑。

## 已完成内容

### 1. 升级数据丢失根因

- `tool/build_windows.ps1` 曾把整个构建目录 `build/.../data` 复制到安装器暂存目录。
- 自动化测试会在该目录生成 `material-center.sqlite` 空业务数据库。
- Inno Setup 又递归安装整个暂存目录，因此更新安装时空数据库覆盖了安装目录中的用户数据库。
- 被覆盖后的主库与旧 WAL 不匹配，进一步出现 `database disk image is malformed`。
- 本机 `material-center.sqlite.pre-v11.bak` 完整性检查通过，包含 228 条素材、227 条分析结果、14238 条帧分析、129 条任务、545 条维度数据。
- 恢复前的安装目录数据库已复制到 `backup/v0.1.176/runtime-db-before-v0.1.153-recovery`，未修改原始证据。

### 2. 数据库与安装器修复

- 构建脚本只复制 `data/models`，不再复制整个 `data`。
- 暂存目录新增数据库文件扫描；发现 `.sqlite`、`.sqlite3`、`.db`、WAL/SHM 或向量索引数据会直接终止打包。
- Inno Setup 主文件规则排除 `data`，仅显式安装模型到 `data/models`。
- 全局数据库固定写入 `%LOCALAPPDATA%/DIT Tools/影资管家/data`，安装目录仅保留只读模型。
- 新增旧数据库恢复流程：只读检查候选库、按业务数据量选取最佳候选、`VACUUM INTO` 一致性快照、校验后原子替换、异常目标隔离和一次性迁移标记。
- 若目标库已有业务数据则绝不覆盖；若恢复失败则保留原始文件并明确报错。

### 3. 快捷搜索结果卡片修复

- 根因是 QML delegate 使用了 `required property` 后仍通过 `model.xxx` 读取角色，Qt 6 不再隐式注入这些角色，导致文本和图片绑定全部失效。
- 素材与帧列表模型统一增加 `quickPreviewPath`、`quickDetail`、`quickMeta`、`quickReasons` 角色。
- 卡片现在显示真实缩略图、帧时间、文件名、项目/来源、摘要、类型、分析状态、匹配原因和排名。
- 图片素材无专用缩略图时回退到原图。
- 点击结果会选择素材、加载详情并进入主页面素材中心；帧搜索仅命中帧而列表无素材时也能同步加载详情。

### 4. 快捷窗口拖拽与位置记忆

- 顶部增加可拖拽区域和可视拖拽柄。
- 释放时写入 AppSettings，下一次打开自动恢复位置。
- 支持负坐标多显示器布局；显示器变化或保存位置越界时自动夹取到可见区域。

### 5. 已完成验证

- 数据恢复回归测试通过：空主库 + 有数据的旧备份可恢复；一次性标记阻止用户主动清空后被旧备份复活。
- AppSettings、QuickSearchController、MaterialCenter UI 契约测试通过。
- Windows MSVC Release 主程序构建成功，QML 缓存生成成功。
- 全量 CTest：28/28 通过。
- `git diff --check` 通过，仅有既有的 CRLF 提示。

## 关键代码前后对比

### 安装数据

修改前：复制整个 `build/data`，安装器递归写入 `{app}/data`。

修改后：只复制模型目录，任何数据库文件进入安装器暂存区都会导致构建失败；用户数据库固定存入 LocalAppData。

### 快捷搜索角色绑定

修改前：delegate 只声明 `required property int index`，其余字段通过 `model.fileName` 等读取。

修改后：delegate 显式声明全部 `required property` 并直接绑定角色名，避免 Qt 6 角色注入规则导致空白卡片。

### 详情打开

修改前：只有素材列表存在对应条目时才能打开详情，帧独立命中可能失效。

修改后：缺少缓存时同步拉取素材详情，成功后再进入素材中心；失败则不跳入空详情页。

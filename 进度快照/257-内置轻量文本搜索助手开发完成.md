# 257 - 内置轻量文本搜索助手开发完成

## 任务目标

为影资管家的素材搜索内置一个不解析图片、仅辅助理解用户文字意图的轻量本地模型。目标是在无独立显卡或 6 GB 显卡设备上也可用，并且只在本地规则/混合检索结果不可靠时按需启动。

## 当前模块

模块已完成，当前版本为 `v0.1.157`，等待用户验收或提出下一轮调整。

## 已完成内容

### 1. 完整方案与边界

- 新增 `dit-tools-src/cinevault-pro/docs/内置轻量文本搜索助手完整方案.md`。
- 明确模型只接收搜索文字，不接收图片、视频、数据库行、文件路径或候选素材 ID。
- 日期、类型、结果域等硬条件始终由确定性本地代码计算并优先。
- 首轮搜索不等待模型；只有空结果、弱结果或复杂口语等低可靠性情况才启动模型。

### 2. 日期与可靠性门控

- `NaturalLanguageQueryParser.cpp` 增加“一个月前 / 一个月之前 / 最近一个月 / 近一个月”等自然月语义。
- 新增 `SearchReliabilityEvaluator.h/.cpp`，根据结果数量、首项分数、置信度、词法/实体证据和复杂表达判断是否需要助手。
- `MaterialCenterQueryService` 将可靠性评估随搜索结果返回，数据库或检索引擎不可用时禁止无意义模型补救。

### 3. 本地模型运行时

- 模型：`Qwen3-0.6B-Q8_0.gguf`，639446688 字节，Apache-2.0。
- 运行时：llama.cpp `b10012` Windows x64 CPU 版。
- 默认 `--n-gpu-layers 0`、4096 上下文、单并发、禁用思考，只监听 `127.0.0.1` 随机端口。
- Windows 子进程使用 `CREATE_NO_WINDOW`，不会弹出控制台窗口。
- 支持启动探活、30 秒超时、重复启动幂等、异常后重试和应用退出清理。

### 4. 查询协议与安全合并

- 新增 `SearchAssistantClient.h/.cpp`，调用本机 OpenAI 兼容接口。
- 固定严格 JSON Schema、低温度、短输出和提示词注入隔离规则。
- 小模型返回的日期在客户端被清空，最终日期只能来自 Qt 本地解析器。
- 已有严格实体时，模型不能增加另一个不同实体作为额外 AND 硬条件；不同标签只可贡献词法补充。
- 模型非法、低置信度、超时或资产缺失时保留首轮本地结果。

### 5. 搜索编排与界面

- `MaterialCenterViewModel` 改为“首轮本地搜索 -> 可靠性评估 -> 必要时模型理解 -> 安全合并后二次搜索”。
- 查询缓存升级为 `v3`，并保留查询代次检查，旧异步结果不能覆盖新查询。
- 内置文本助手与远程候选帧视觉复核完全分离。
- “仅本地搜索”允许内置文本助手，但禁止远程帧复核。
- 设置页显示模型是否缺失、启动中、已就绪或失败回退。

### 6. 构建与发布

- 新增 `cmake/search-assistant-dependencies.lock.json` 与 `PrepareSearchAssistantDependencies.cmake`。
- 下载资产按固定 URL、大小和 SHA-256 校验，缓存位于源码树外。
- RealWorkflow/FFmpeg 预设自动启用并打包助手；运行时与模型缺一时安装包构建直接失败。
- 修复干净完整构建中 `MaterialCenterQueryServiceTest` 遗漏可靠性评估器链接的问题。
- 生成安装包 `output/v0.1.157/CineVault-Setup-v0.1.157.exe`。

## 关键代码前后对比

### 搜索触发

修改前：

```cpp
executeSearch();
startSearchUnderstanding(m_lastParsedQuery);
```

模型辅助只看设置，不评估首轮结果质量，并依赖远程视觉接口。

修改后：

```cpp
executeSearch();
startSearchUnderstanding(m_lastParsedQuery, m_lastSearchReliability);
```

只有 `reliability.shouldUseAssistant` 为真时才按需启动本地 Qwen3 文本模型。

### 日期保护

修改前：模型查询计划可携带日期进入合并。

修改后：

```cpp
understanding->dateConstraint = {};
```

模型只辅助识别含义，不能引入或覆盖日期过滤。

### Windows 后台进程

修改前：启动控制台型 `llama-server.exe` 可能出现窗口闪烁。

修改后：

```cpp
arguments->flags |= CREATE_NO_WINDOW;
```

助手作为无窗口本地子进程运行。

# 259 - 实时模型搜索、快捷筛选与中键拖动完成并发布 v0.1.159

## 任务目标

完成三项搜索体验增强：

1. 本地轻量语言模型实时介入自然语言查询，统一“帧/画面”语义，并避免“男人穿牛仔裤”退化为只命中牛仔裤。
2. 主界面和快捷搜索框增加“智能 / 视频 / 帧画面 / 文档”快捷筛选。
3. 搜索结果、详情和快捷搜索列表支持鼠标中键按住拖动滚动。

## 当前模块

全部开发、测试、Release 构建、安装包生成和清理均已完成，版本为 `v0.1.159`。

## 已完成内容

### 1. 实时本地模型查询理解

- 首轮本地规则、FTS5、结构化事实和 BGE 搜索保持即时执行，不等待模型。
- 每次包含内容语义的稳定输入在 400ms 防抖后调用本地 Qwen3 0.6B，不再由 `shouldUseAssistant` 可靠性门禁决定是否介入。
- 保留查询缓存、并发 generation 校验、模型失败时保留首轮结果等安全机制。
- 设置文案由“按需使用”改为“实时使用内置轻量文本模型辅助理解查询”。

### 2. 帧与画面同义归一化

- 无显式视频、图片或文档目标时，“画面”与“帧”都解析为 `SearchResultTarget::Frames`。
- “画面文字写着……的视频”仍返回视频素材，避免把“画面文字”误当帧结果目标。
- `有男人穿着牛仔裤的帧` 与 `有男人穿着牛仔裤的画面` 得到相同结果域和内容语义。

### 3. 多实体同帧约束

- `ParsedMaterialQuery` 增加 `explicitEntityLabels`，记录用户原文明确出现的实体，和带修饰词的 `strictEntities` 分离。
- 多实体语句不再由规则层武断地把颜色、材质绑定到第一个名词，交给文本模型解析。
- 模型提示明确要求“男人穿牛仔裤”输出男人和牛仔裤两个 entity。
- 模型输出再由原文实体事实校正：当 0.6B 模型把多个对象折叠成一个对象时，恢复为独立、原文可验证的必需实体。
- 查询服务继续用 `factsMatchAllConstraints` 要求所有实体在同一帧事实中共现，因此只有牛仔裤、没有男人的帧会被排除。

### 4. 搜索命中快捷筛选

- 新增 `SearchResultQuickFilter`：`Smart / Video / Frames / Document`。
- `MaterialSearchScope` 将筛选传到 `SearchEngine`，在自然语言与模型解析之后覆盖结果域和素材类型。
- 主界面“搜索命中”栏和快捷搜索框均显示四个按钮，选中状态共享并实时同步。
- 点击“帧画面”会真正重跑帧查询；点击“文档”会重跑文档查询，不是仅隐藏当前列表项。
- 快捷筛选与顶部素材类型下拉框保持可预测联动：点击快捷按钮同步对应素材类型；手动修改类型下拉框回到“智能”。

### 5. 鼠标中键拖动

- 新增 `MiddleDragScrollHandler.qml` 可复用组件。
- 组件只接受 `Qt.MiddleButton` 和鼠标设备，`target: null`，不会移动视图控件本身。
- 按下时取消惯性滚动并记录 `contentX/contentY`；拖动时更新内容位置并限制边界；松开时回弹到合法范围。
- 已接入主界面帧网格、文件夹结果、素材结果、详情、批量解析弹窗、维度列表，以及快捷搜索的文件夹/素材结果列表。

## 关键代码前后对比

### 模型调用门禁

修改前：

```cpp
if (!reliability.shouldUseAssistant) {
    return;
}
```

修改后：

```cpp
executeSearch();
startSearchUnderstanding(m_lastParsedQuery);
```

除空查询和纯日期/类型等无内容条件外，稳定输入都会调用本地文本助手；首轮搜索仍先显示。

### 画面目标

修改前：

```cpp
query.frameIntent = !query.folderIntent && working.contains(QStringLiteral("帧"));
```

修改后：

```cpp
query.frameIntent = !query.folderIntent
    && (hasFrameWord || hasStandalonePictureIntent);
```

`hasStandalonePictureIntent` 会避开“画面文字/画面内容/画面描述”和显式素材类型。

### 快捷筛选

修改前：仅由自然语言 `resultTarget` 决定返回素材、文件夹或帧。

修改后：

```cpp
applyResultQuickFilter(&result.parsedQuery, scope.resultQuickFilter);
```

快捷筛选在本地与模型理解之后成为最终用户显式选择。

### 中键拖动

```qml
DragHandler {
    acceptedButtons: Qt.MiddleButton
    target: null
    onTranslationChanged: {
        flickable.contentX = boundedContentX(startContentX - activeTranslation.x)
        flickable.contentY = boundedContentY(startContentY - activeTranslation.y)
    }
}
```

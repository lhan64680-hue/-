# 203-旧素材源全文件补扫

- 时间：2026-07-05
- 当前模块：打开已有项目后的旧素材源全文件补扫

## 已完成内容

- 已读取上一份最新快照：`进度快照/202-素材中心全文件筛选.md`。
- 已创建阶段备份：`backup/v0.1.156/`。
- 已补充本模块代码对比文档：`backup/v0.1.156/素材中心旧索引补扫-代码对比.md`。
- 已确认原因：顶部下拉和素材中心查询层已经默认“全部文件”；旧项目打开后仍只显示视频，主要是功能上线前添加的素材目录在项目库 `asset_file` 中只有旧扫描时代的视频记录，全局素材中心同步不到不存在的非视频记录。
- 项目库 `source_root` 新增 `scan_version` 字段，并通过旧库兼容迁移自动补齐。
- 扫描完成后写入 `ScanEngine::CurrentScanVersion`，标记该素材源已经完成全文件扫描。
- 打开项目后自动检查 `scan_version` 落后的历史素材源，路径存在且可读时后台补扫一次。
- 补扫复用原有扫描完成后的 `catalogChanged -> syncCurrentProject` 同步链路，补扫完成后素材管理中心会显示视频、音频、图片、文档等全部文件。
- 新增 `ImportServiceLegacyRescanTest`，覆盖旧项目只有视频索引、实际目录包含 `mov/wav/md` 时自动补扫为 3 条全文件记录。
- 已完成 Release + FFmpeg 构建并生成新安装包。
- 已清理临时构建日志和空的 `_installer_staging` 目录。

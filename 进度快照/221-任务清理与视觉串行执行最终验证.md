## 具体修改的代码前后对比

### 1. 清理已完成任务

修改前：任务引擎只有 `clearJobs()`，会清空全部任务，不适合作为页面按钮。

```cpp
void clearJobs();
```

修改后：新增只清理 `Completed` 的方法。

```cpp
void clearJobs();
void clearCompletedJobs();
```

核心逻辑：

```cpp
for (const auto &job : m_jobs) {
    if (job.state != JobState::Completed) {
        keptJobs.append(job);
    }
}
```

任务页 QML 新增：

```qml
ActionButton {
    text: "清理已完成"
    enabled: viewModel && viewModel.canClearCompletedJobs
    onClicked: if (viewModel) viewModel.clearCompletedJobs()
}
```

### 2. 多维度解析改为排队

修改前：`analyzeDimensions()` 校验后立即 `QtConcurrent::run(...)`，批量多维度解析会同时启动多个后台任务。

修改后：公开入口只入队并提示等待。

```cpp
m_dimensionAnalysisKeys.insert(normalizedKey);
m_dimensionAnalysisQueue.enqueue(DimensionAnalysisJob{normalizedKey, requestedDimensions});
emit dimensionAnalysisProgressChanged(normalizedKey, true, QStringLiteral("等待多维度解析队列"), QString());
startNextAnalysis();
```

原执行体保留为私有 `startDimensionAnalysisNow(...)`。

### 3. 统一视觉任务执行槽

修改前：基础解析队列和多维度解析互不约束。

修改后：`startNextAnalysis()` 优先处理基础解析队列；基础队列为空时才启动下一条多维度解析。

```cpp
if (m_analysisQueue.isEmpty()) {
    if (!m_dimensionAnalysisQueue.isEmpty()) {
        startNextDimensionAnalysis();
        return;
    }
    emit analysisQueueChanged(QString(), 0);
    return;
}
```

多维度解析启动时设置 `m_analysisRunning = true`，结束时释放并调度下一条：

```cpp
m_analysisRunning = false;
emit analysisQueueChanged(m_currentVideoKey, m_analysisQueue.size());
startNextAnalysis();
```

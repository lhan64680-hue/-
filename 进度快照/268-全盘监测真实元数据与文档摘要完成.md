# 268 - 全盘监测、真实元数据与文档摘要完成

## 本轮目标

完成总体方案的模块 3、4、5、6：支持直接添加磁盘卷，全盘递归索引；持续监测文件变化并在系统空闲 30 分钟后无感更新；捆绑 ExifTool 读取视频、音频、图片的真实嵌入元数据；让文档正文自动进入摘要与搜索链路。

## 已完成内容

### 1. 磁盘卷与全盘索引入口

- 新增 `StorageVolumeService`，枚举系统已挂载且可用的本地盘、移动盘和网络盘。
- “添加素材源”弹窗新增磁盘卷列表，显示卷标、盘符、磁盘类型、文件系统、可用/总容量和只读状态。
- 盘符根目录可以直接作为素材源，复用原子扫描管线完成全盘递归索引。
- 文件系统遍历使用 `skip_permission_denied`，权限受限目录计入警告而不会破坏已存在的索引。

### 2. 递归变化监测与系统空闲调度

- 新增 `SourceChangeMonitor`：Windows 使用异步 `ReadDirectoryChangesW`，递归监测每个素材源，支持深层目录变化、停止事件和 1.5 秒防抖。
- 新增 `SystemIdleMonitor`：通过 `GetLastInputInfo` 读取全系统键鼠空闲时长，而不是只判断本软件窗口。
- 新增 `BackgroundMaintenanceCoordinator`：
  - 项目打开后自动监测所有素材源；
  - 变化只标记脏代次，不立即抢占磁盘；
  - 系统连续空闲 30 分钟后按素材源串行重扫；
  - 同一时刻只派发一个 AI 解析任务；
  - 检测到用户恢复操作后不再派发下一项；
  - 项目重新打开时把素材源标记为待核对，补偿软件关闭期间的变化。
- `BackgroundMonitoringTest` 已覆盖空闲阈值、恢复操作和真实嵌套文件变化通知。

### 3. ExifTool 真实嵌入元数据

- 固定并校验官方 ExifTool 13.59 Windows x64 运行时：
  - 锁文件：`cmake/exiftool-dependencies.lock.json`；
  - 准备脚本：`cmake/PrepareExifToolDependency.cmake`；
  - SHA-256：`44b512b25af500724ba579d0a53c8fc5851628b692dd5e5d94ae4a15c2cba9ec`。
- Release 构建后自动把 `exiftool.exe` 和 `exiftool_files` 复制到程序目录的 `exiftool/`，安装器暂存也会校验并携带运行时。
- 项目数据库升级到版本 5，新增 `embedded_metadata`：保存拍摄/创建时间、相机和镜头、脱敏序列号指纹、GPS、方向、尺寸、时长、帧率、编码、色彩、采样率、声道、码率、时间码、标题、描述、艺术家、专辑、流派、关键词、检索文本和脱敏后的原始 JSON。
- `ExifToolAdapter` 使用批量 JSON、数字原值、QuickTime UTC 和大文件支持参数；每批 32 个文件。
- `MetadataExtractionService` 使用独立只读任务上下文和事务批量落库，只处理新增/发生变化的音视频图片；失败的未变化文件不会在每次重扫中反复消耗资源。
- 设备序列号只保存不可逆 SHA-256 派生指纹，原始 JSON 和搜索文本中的序列号被替换为脱敏值。
- 新增真实 JPEG 回归和后台写库端到端回归，验证运行时、字段归一化、脱敏、线程连接和 SQL 参数数量。

### 4. 元数据详情与全字段搜索

- 全局数据库升级到版本 13，新增 `embedded_metadata_text`；旧语义索引自动标记为待重建。
- 项目素材列表关键词搜索、素材管理中心词法搜索、文件夹包含素材搜索、FTS 和语义搜索文档均纳入 ExifTool 元数据。
- 素材检查器新增 ExifTool 状态/版本、相机、镜头、GPS、尺寸、帧率、编码、色彩、音频、时间码和内容标签；提示完整脱敏 JSON 已保存在项目数据库。
- 拍摄时间优先采用 ExifTool 的真实嵌入时间，无法取得时再回退到 ffprobe、目录日期和文件修改时间。
- 修复刷新 FTS 时可能覆盖已有 AI 摘要、关键词与逐帧文字的问题：重建 FTS 行时会重新合并解析结果表和帧表。

### 5. 文档自动摘要闭环

- PDF 正文通过 Qt PDF 逐页提取，最多保留 64,000 字；加密 PDF、不可解析 PDF 和扫描图片 PDF 会返回明确错误提示。
- PDF 与 TXT、LOG、Markdown、JSON、CSV/TSV、XML/YAML、DOCX/XLSX/PPTX、SRT/ASS/VTT 一起进入自动摘要队列。
- 正文提取后先写入可搜索 `source_text`，即使模型摘要失败，文档正文仍可被搜索。
- 系统空闲自动解析优先处理文档和字幕，再处理视频/图片，避免大量视频长期阻塞文档摘要。
- 历史 `IndexedOnly` PDF 在下次目录同步时自动升级为 `Pending`，无需重新导入。

### 6. 性能优化

- 媒体技术参数和缩略图任务不再根据素材源总数整批重跑；只为数据库中缺失结果的新增/变化素材创建任务。
- 未变化且已成功的素材不会产生空任务、重复 ffprobe 或重复缩略图生成。
- 原子扫描、媒体探测/缩略图和 ExifTool 工作线程使用可恢复的 Windows `BELOW_NORMAL` 优先级，完成后恢复线程池原优先级。

## 具体修改前后对比

### 自动监测

修改前：

```text
用户手动点击重扫 -> 才能发现新增/删除/修改文件
```

修改后：

```text
ReadDirectoryChangesW 递归通知 -> 素材源脏代次 +1
系统空闲达到 30 分钟 -> 串行原子重扫 -> 真实元数据/媒体任务 -> 自动摘要
用户恢复操作 -> 停止派发后续任务
```

### 媒体元数据

修改前：

```text
ffprobe 技术流信息；每次重扫可能把整个素材源重新处理
```

修改后：

```text
ffprobe 技术流信息 + ExifTool 嵌入元数据
只处理 embedded_metadata/media_metadata/thumbnail 缺失的新增或变化素材
```

### 文档摘要

修改前：

```text
PDF 仅进入元数据搜索；文档需人工触发解析
```

修改后：

```text
PDF/Office/纯文本正文提取 -> 正文立即可搜索
空闲调度优先自动生成文档摘要 -> 结果自动生效
```

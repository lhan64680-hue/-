# 264 - 搜索召回保护与快捷定位拖拽修复完成

## 任务目标

1. 修复 Qwen3 0.6B 介入后把原本有结果的向量搜索压成空结果。
2. 快捷搜索素材卡片双击后，跨工程打开目标工程，进入素材管理中心并精确高亮目标卡片，同时强制关闭快捷搜索框。
3. 修复快捷搜索窗口拖拽漂移和不跟手。

## 已完成内容

### 模块 1：小模型增强的召回保护

- `SearchQueryUnderstanding.cpp`
  - 本地解析已有语义文本时，不再把模型改写文本拼接进去，避免向量查询被稀释。
  - 模型新增的日期、素材类型、OCR、实体和实体属性只有在用户原始输入中有明确文本依据时，才允许升级为硬筛选条件。
  - 保留显式别名映射，例如“女人/女性/女子/女士”和“牛仔裤/裤子/长裤/丹宁裤”。
- 新增 `SearchResultFusion.h/.cpp`
  - 模型增强结果为空时保留基础向量/本地结果。
  - 模型擅自改变结果类型时保留基础结果类型。
  - 同类型增强结果按模型排序在前，并补回模型漏掉的基础召回项。
- `MaterialCenterViewModel.cpp`
  - 每次查询保存同一 generation 的基础结果，模型返回后通过 `SearchResultFusion` 合并，不再直接覆盖。

关键修改前后：

```cpp
// 修改前：本地没有严格实体时，模型返回的任意实体都可成为硬条件
if (!localHasStrictEntities || matchesKnownEntity || queryExplicitlyRequiresEntity(...)) {
    mergeEntity(...);
}

// 修改后：模型实体必须匹配本地已知实体，或在用户原文中明确出现
if (matchesKnownEntity || queryExplicitlyRequiresEntity(...)) {
    mergeEntity(...);
}
```

```cpp
// 修改前：模型增强搜索结果直接覆盖基础结果
applySearchResult(enhancedResult);

// 修改后：增强结果必须保留基础召回
auto outcome = SearchResultFusion::preserveBaselineRecall(
    m_lastBaselineSearchResult, enhancedResult);
applySearchResult(outcome.result);
```

### 模块 2：快捷搜索跨工程精确定位

- `MaterialCenterViewModel.h/.cpp`
  - 新增持久化定位状态：目标素材 ID、目标帧号、目标索引和 revision。
  - `openQuickSearchResultAtIndex()` 在切换工程前保存快捷查询和被双击帧号；打开工程后恢复查询、限定目标工程、重新加载并选择目标素材。
  - 同一素材命中多帧时，根据被双击结果索引保存精确帧号，不再错误高亮第一帧。
- `QuickSearchWindow.qml`
  - 素材和帧卡片双击调用带索引的定位入口。
  - 成功进入素材中心或打开失败回主窗口时，都使用强制关闭参数；即便快捷框处于“固定显示”状态也会关闭。
- `MaterialCenterWorkspace.qml`
  - 工作区加载、变为可见、搜索状态更新时都会尝试恢复定位。
  - 自动滚动到目标卡片中央。
  - 目标素材/帧卡片显示 3.2 秒蓝色加粗边框、浅蓝背景和脉冲内环。

关键修改前后：

```qml
// 修改前：固定状态可能阻止快捷框关闭
function showMainWindow() {
    hideSearch()
}

// 修改后：双击进入工程时显式强制关闭
function showMainWindow(forceCloseQuickSearch) {
    if (forceCloseQuickSearch || !pinned) {
        hideSearch()
    }
}
```

### 模块 3：原生无漂移窗口拖拽

- `QuickSearchWindow.qml`
  - 删除 `dragOrigin + DragHandler.translation` 手工计算窗口坐标的逻辑。
  - 改用 `Window.startSystemMove()`，由 Windows/Qt 处理 DPI、鼠标捕获和窗口移动。
  - 拖拽结束后再保存受屏幕范围约束的位置。

关键修改前后：

```qml
// 修改前：窗口坐标变化反向影响 DragHandler translation，产生漂移
root.x = root.dragOrigin.x + translation.x
root.y = root.dragOrigin.y + translation.y

// 修改后：使用系统原生窗口移动
root.startSystemMove()
```

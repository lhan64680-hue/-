## 关键代码前后对比

### 1. 结构化事实验证粒度

修改前：

```cpp
QHash<QString, QVector<VisionEntityFact>> completeFactsByAsset;
completeFactsByAsset[videoKey].append(entities);
factsMatchAllConstraints(completeFactsByAsset.value(videoKey), constraints);
```

同一素材或文件夹的全部帧实体被扁平合并，多个条件可能由不同帧分别满足。

修改后：

```cpp
QHash<QString, QVector<QVector<VisionEntityFact>>> completeFactsByAssetFrame;
completeFactsByAssetFrame[videoKey].append(entities);
anyFrameFactsMatchAllConstraints(completeFactsByAssetFrame.value(videoKey), constraints);
```

每帧事实独立保存，只有单帧满足全部严格条件才算命中。

### 2. 旧数据文本证据

修改前：

```cpp
evidence.caption = frame.caption.trimmed().isEmpty()
    ? frame.ocrText.trimmed()
    : frame.caption.trimmed();
```

修改后：

```cpp
evidence.caption = matchingFrameEvidenceText(frame, constraints);
```

只组合真正包含查询条件的同帧描述、标签、对象、实体或 OCR 字段。

### 3. 批量策略弹窗

修改前：固定高度和固定选项卡高度，窄/短窗口可能裁切，居中坐标也可能为负数。

修改后：

```qml
width: Math.max(1, Math.min(620, root.width - 24))
height: Math.max(1, Math.min(430, root.height - 24))
x: Math.max(0, Math.round((root.width - width) / 2))
y: Math.max(0, Math.round((root.height - height) / 2))
```

对话框内容置于 `ScrollView`，选项卡使用内容高度，所有操作在小视口仍可访问。

# 220-视觉解析任务串行执行

## 已完成内容

- 已完成第二个功能模块：将多维度解析纳入 `VideoAnalysisService` 的统一视觉任务调度，不再为每个素材直接并发启动后台解析。
- 基础内容解析原本已有 `m_analysisQueue + m_analysisRunning` 串行机制；本轮新增多维度解析队列，并让两类任务共用同一个执行槽。
- 批量多维度解析时，每个素材先进入队列，状态提示为“等待多维度解析队列”；当前任务完成或失败后自动启动下一条。
- 同一素材如果已经在基础解析队列、多维度解析队列或正在执行，会统一提示“该素材已在视觉解析队列中”，避免重复入队。
- 已构建验证：`cmake --build --preset windows-msvc-release-ffmpeg --config Release --target CineVault` 通过。

## 当前修改到哪个模块

- 串行调度核心：`src/application/VideoAnalysisService.h/.cpp`

## 具体修改的代码前后对比

### 1. 多维度解析由直接并发改为入队

修改前：`analyzeDimensions()` 校验通过后立即创建任务并 `QtConcurrent::run(...)`，批量循环调用时会同时启动多个多维度解析。

```cpp
m_dimensionAnalysisKeys.insert(normalizedKey);
emit dimensionAnalysisProgressChanged(normalizedKey, true, QStringLiteral("准备多维度解析"), QString());

auto future = QtConcurrent::run([this, normalizedKey, requestedDimensions, config, jobId]() {
    // 执行多维度解析
});
```

修改后：公开入口只负责校验和入队。

```cpp
m_dimensionAnalysisKeys.insert(normalizedKey);
m_dimensionAnalysisQueue.enqueue(DimensionAnalysisJob{normalizedKey, requestedDimensions});
emit dimensionAnalysisProgressChanged(normalizedKey, true, QStringLiteral("等待多维度解析队列"), QString());
startNextAnalysis();
return true;
```

原来的执行体保留为私有 `startDimensionAnalysisNow(...)`，避免重写复杂的逐帧多维度解析逻辑。

### 2. 基础解析和多维度解析共用一个执行槽

修改前：`startNextAnalysis()` 只看基础解析队列；多维度解析不受该队列约束。

```cpp
if (m_analysisQueue.isEmpty()) {
    emit analysisQueueChanged(QString(), 0);
    return;
}
```

修改后：基础解析队列为空时，再启动下一条多维度解析。

```cpp
if (m_analysisQueue.isEmpty()) {
    if (!m_dimensionAnalysisQueue.isEmpty()) {
        startNextDimensionAnalysis();
        return;
    }
    emit analysisQueueChanged(QString(), 0);
    return;
}
```

`startNextDimensionAnalysis()` 会设置 `m_analysisRunning = true`，因此基础解析和多维度解析任意时刻只会有一个任务在执行。

### 3. 多维度任务结束后自动推进下一条

修改前：多维度解析结束只移除当前素材 key 并发出进度/目录刷新信号。

修改后：结束时释放全局执行槽，并继续调度下一条任务。

```cpp
m_dimensionAnalysisKeys.remove(normalizedKey);
emit dimensionAnalysisProgressChanged(normalizedKey, false, detail, error);
if (success) {
    emit catalogChanged();
}
m_analysisRunning = false;
emit analysisQueueChanged(m_currentVideoKey, m_analysisQueue.size());
startNextAnalysis();
```

### 4. 重复入队检查统一到视觉解析队列

修改前：基础解析只检查基础队列，多维度解析只检查多维度运行 key。

修改后：同一素材只要在任一视觉解析队列中，就会被拒绝重复入队。

```cpp
if (m_currentVideoKey == normalizedKey
    || m_queuedVideoKeys.contains(normalizedKey)
    || m_dimensionAnalysisKeys.contains(normalizedKey)) {
    *errorMessage = QStringLiteral("该素材已在视觉解析队列中。");
    return false;
}
```

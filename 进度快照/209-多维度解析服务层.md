# 209-多维度解析服务层

- 时间：2026-07-05
- 当前模块：素材管理中心多维度解析服务层

## 已完成内容

- 已在 `VideoAnalysisService` 新增 `analyzeDimensions(videoKey, dimensions)` 服务入口。
- 已新增多维度解析进度信号 `dimensionAnalysisProgressChanged`。
- 已实现服务层校验：
  - 必须选择素材。
  - 视觉接口配置必须完整。
  - 素材必须已完成基础解析。
  - 维度列表必须非空。
  - 同一素材不能重复启动多维度解析任务。
  - 已存在维度会被过滤，不重复请求模型。
- 已实现基础上下文组装：
  - 文件名、素材类型、技术摘要。
  - 基础摘要、关键词、场景。
  - 文本/文档正文预览。
  - 视频逐帧解析节选。
  - 已有补充维度，用于提示模型不要重复。
- 已实现模型调用与落库：
  - 调用 `VisionApiClient::analyzeDimensions`。
  - 保存到 `material_dimension_analysis`。
  - 使用 `video_key + dimension_key` 去重。
  - 完成后刷新素材管理中心目录数据。
- 已接入任务中心 `JobEngine`，多维度解析会作为内容解析任务显示。

## 具体修改的代码前后对比

详细阶段对比将在 `backup/v0.1.160/多维度解析功能-代码对比.md` 中汇总。

修改前：

```cpp

```

修改后：

```cpp
bool retryFrame(const QString &videoKey, int frameNumber, QString *errorMessage = nullptr);

```

### 进度信号

修改前：只有基础内容解析进度信号。

修改后：

```cpp
void dimensionAnalysisProgressChanged(const QString &videoKey,
                                      bool running,
                                      const QString &detail,
                                      const QString &errorMessage);
```

### 去重策略

修改前：没有维度补充任务。

修改后：

```cpp
const auto pendingDimensions = filterNewDimensions(requestedDimensions, existingDimensions);
if (pendingDimensions.isEmpty()) {
    finish(false, "所选维度已存在。", "所选维度已存在，不重复解析。");
    return;
}
```

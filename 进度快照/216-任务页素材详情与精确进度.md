## 具体修改的代码前后对比

### 任务数据结构

修改前：`Job` 只有标题、详情、百分比、状态等轻量字段，无法知道对应素材、缩略图和精确步骤。

```cpp
struct Job {
    qint64 id = 0;
    JobType type = JobType::Scan;
    JobState state = JobState::Pending;
    QString title;
    QString detail;
    QString errorMessage;
    qint64 progress = 0;
    qint64 sourceRootId = 0;
    QDateTime startedAt;
    QDateTime updatedAt;
};
```

修改后：新增运行期对象和结构化进度，不落库、不迁移旧数据库。

```cpp
struct JobSubject {
    QString kind;
    QString key;
    QString name;
    QString path;
    QString thumbnailPath;
    QString typeLabel;
};

struct JobProgressContext {
    int currentStep = 0;
    int totalSteps = 0;
    QString stepLabel;
    qint64 currentItem = 0;
    qint64 totalItems = 0;
    QString unitLabel;
    int currentFrameNumber = 0;
    QString extraLabel;
};

struct Job {
    ...
    JobSubject subject;
    JobProgressContext progressContext;
    QDateTime startedAt;
    QDateTime updatedAt;
};
```

### JobEngine 更新

修改前：只能更新百分比和详情文案。

```cpp
void JobEngine::updateJob(qint64 jobId, qint64 progress, const QString &detail)
{
    job->progress = progress;
    job->detail = detail;
}
```

修改后：支持结构化进度；旧调用不传 context 时不会清空已有结构化进度。

```cpp
void JobEngine::updateJob(qint64 jobId,
                          qint64 progress,
                          const QString &detail,
                          const JobProgressContext &progressContext)
{
    job->progress = progress;
    job->detail = detail;
    if (hasProgressContext(progressContext)) {
        job->progressContext = progressContext;
    }
}
```

### 多维度逐帧进度

修改前：逐帧多维度解析期间任务百分比长期固定为 35%。

```cpp
auto report = [&](const QString &detail) {
    updateJob(jobId, 35, detail);
};
```

修改后：按帧请求推进百分比，并记录当前步骤、帧序号和实际帧号。

```cpp
const auto frameProgress = 20 + (static_cast<qint64>(frameRequestIndex) * 55) / frameTotal;
report(QStringLiteral("逐帧多维度解析 %1/%2：第 %3 帧")
           .arg(frameRequestIndex)
           .arg(frameTotal)
           .arg(frame.frameNumber),
       frameProgress,
       analysisProgressContext(2,
                               4,
                               QStringLiteral("逐帧多维度解析"),
                               frameRequestIndex,
                               frameTotal,
                               QStringLiteral("帧"),
                               frame.frameNumber));
```

### 右侧任务详情

修改前：右侧面板主要显示当前解析批次，不跟随左侧选中任务。

```qml
readonly property bool hasCurrentItem: viewModel && viewModel.hasActiveBatchItem
text: hasCurrentItem ? viewModel.batchCurrentLabel : "暂无正在处理的素材"
```

修改后：右侧面板跟随 `selectedJobId`，显示选中任务对象、缩略图/占位、结构化进度、状态记录，并保留批次总览。

```qml
readonly property bool hasSelectedTask: viewModel && viewModel.hasSelection
readonly property string selectedThumbnailPath: hasSelectedTask ? viewModel.selectedSubjectThumbnailPath : ""
text: hasSelectedTask ? fallbackText(viewModel.selectedSubjectName, viewModel.selectedJobTitle) : "选择左侧任务查看详情"
```

### 扫描后台测试稳定性

修改前：测试收到 `scanFinished` 后可能还有后台扫描 future 未完全收尾。

```cpp
QVERIFY(scanFinished.wait(10000));
QCOMPARE(countAssets(databaseManager.database(), &errorMessage), qint64{3});
```

修改后：测试显式等待扫描 future 完全结束并排空事件。

```cpp
QVERIFY(scanFinished.wait(10000));
scanEngine.waitForIdle();
QCoreApplication::processEvents();
QCOMPARE(countAssets(databaseManager.database(), &errorMessage), qint64{3});
```

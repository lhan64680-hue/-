# 238 - 视觉 OCR 空值落库修复并完成测试包

## 本轮目标

修复视频逐帧视觉解析在模型响应缺失 `ocr_text` 或返回 `null` 时，将 Qt null `QString` 绑定成 SQL `NULL`，继而触发 `video_frame_analysis.ocr_text` 的 `NOT NULL` 约束失败；完成正式 MSVC Release 构建、全量测试和独立测试包部署。

## 已完成内容

1. 在视觉响应解析层将缺失、`null` 或不支持类型的标量文本归一化为非 null 的空字符串。
2. 在帧分析数据库 UPDATE 层增加防御性兜底，确保以下 `NOT NULL` 文本列不会绑定 SQL `NULL`：
   - `entities_json`
   - `ocr_text`
   - `ocr_blocks_json`
   - `model_name`
   - `prompt_version`
   - `analyzed_at`
   - `last_attempt_at`
3. 增加两项回归场景：
   - 模型响应省略 `ocr_text`。
   - 模型响应显式返回 `"ocr_text": null`。
4. 恢复正式构建配置为 `CINEVAULT_ENABLE_FFMPEG=ON` 与 `CINEVAULT_ENABLE_LOCAL_SEARCH=ON`。
5. 完成 MSVC Release 完整构建及 24/24 串行全量测试。
6. 创建新的独立测试包，不覆盖模块 5 原测试包，也不复制旧包的失败数据库和帧任务状态。

## 关键代码前后对比

### 视觉响应空值归一化

文件：`src/infrastructure/network/VisionResponseParser.cpp`

修改前：

```cpp
if (!value.isObject()) {
    return {};
}
```

修改后：

```cpp
if (!value.isObject()) {
    return QStringLiteral("");
}
```

效果：缺失或 `null` 的 `ocr_text` 不再形成 null `QString`。

### 数据库写入兜底

文件：`src/application/VideoAnalysisService.cpp`

修改前：

```cpp
query.addBindValue(frame.ocrText);
query.addBindValue(frame.modelName);
query.addBindValue(frame.promptVersion);
query.addBindValue(frame.analyzedAt);
query.addBindValue(frame.lastAttemptAt);
```

修改后：

```cpp
query.addBindValue(nonNullText(frame.ocrText));
query.addBindValue(nonNullText(frame.modelName));
query.addBindValue(nonNullText(frame.promptVersion));
query.addBindValue(nonNullText(frame.analyzedAt));
query.addBindValue(nonNullText(frame.lastAttemptAt));
```

同时对结构化实体与 OCR 块 JSON 文本进行了同样兜底。

### 回归测试

文件：`tests/unit/VisionApiClientJsonTest.cpp`

新增：

- `normalizeFrameAnalysis_normalizesMissingOcrTextToNonNullEmptyString`
- `normalizeFrameAnalysis_normalizesNullOcrTextToNonNullEmptyString`

两项均断言 `ocrText.isNull() == false` 且 `ocrText.isEmpty() == true`。

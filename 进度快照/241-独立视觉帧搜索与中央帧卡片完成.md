# 241 - 独立视觉帧搜索与中央帧卡片完成

## 一、本轮目标

实现用户提出的独立帧搜索体验：

- 查询示例：`包含了蓝色牛仔裤的帧`。
- 搜索结果必须是帧，而不是把帧折叠为所属素材。
- 每条结果显示帧缩略图和完整视觉信息。
- 帧结果必须显示在中央大操作区，不得复用右侧素材详情栏中的 `selectedFrames` 列表。

## 二、已完成模块

### 模块 1：帧意图与结果协议

涉及文件：

- `dit-tools-src/cinevault-pro/src/domain/SearchTypes.h`
- `dit-tools-src/cinevault-pro/src/core/search/NaturalLanguageQueryParser.cpp`
- `dit-tools-src/cinevault-pro/tests/unit/NaturalLanguageQueryParserTest.cpp`

完成内容：

- `SearchResultTarget` 新增 `Frames`。
- `ParsedMaterialQuery` 新增 `frameIntent`。
- 明确出现“帧/关键帧/视频帧”等词时进入视觉帧结果域；文件夹意图仍有更高优先级，避免“某些帧所在文件夹”错误返回帧。
- `帧、包含了、包含有、包括、带有` 等结果/语法词不会污染视觉内容条件。
- 新增“牛仔裤”实体词；`包含了蓝色牛仔裤的帧` 被清洗为语义内容 `蓝色牛仔裤`。
- 查询解释显示 `目标：视觉帧`。

前后对比：

- 之前：视觉帧语义命中会映射并折叠成素材卡片，最多只附带一个匹配时间点。
- 现在：帧拥有独立结果键 `frame:<videoKey>:<frameNumber>`，同一素材可返回多个不同帧。

### 模块 2：帧级混合召回与严格校验

涉及文件：

- `src/core/search/SearchEngine.cpp`
- `src/application/MaterialCenterQueryService.cpp`
- `src/domain/SearchTypes.h`
- `tests/unit/SearchEngineTest.cpp`
- `tests/unit/MaterialCenterQueryServiceTest.cpp`

完成内容：

- 新增帧级 SQL 词法召回，检索字段包括描述、标签、对象、动作、场景、结构化实体和 OCR。
- BGE/USearch 返回的 `VisualEntity` 文档在帧模式下保持为独立帧，不再映射成素材。
- 日期、项目、素材源、素材类型、解析状态和确认状态继续约束所属素材。
- OCR 意图必须由同一帧的 `ocr_text` 满足，不能由所属素材的其他帧代替。
- 帧结果包含所属素材键、帧号、时间点、描述和混合分数。
- 查询服务装配帧缩略图、文件名、项目、素材源、路径、标签、对象、动作、场景、实体属性、OCR、置信度和命中理由。

严格性规则：

1. 结构化事实完整时，`蓝色 + 牛仔 + 裤子` 必须属于同一帧中的同一视觉实体。
2. `牛仔裤 / 长裤 / 裤子` 在具有牛仔材质约束时按可验证别名处理，但不会把短裤当成长裤。
3. 旧数据结构化事实不完整时，只有同一帧的描述、标签、对象等文本同时包含全部条件才允许低置信度兜底。
4. 结构化事实完整但实体属性不匹配时直接排除，不使用文本绕过。

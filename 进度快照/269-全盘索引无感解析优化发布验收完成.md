## 具体修改前后对比

### 1. 空闲维护避免循环全盘扫描

修改前：

```cpp
markSourceDirty(sourceRoot);
m_attemptedThisIdle.remove(sourceRoot);
```

素材源内持续变化的系统日志可能在同一次空闲期不断清除“已尝试”标记，导致全盘重扫反复开始。

修改后：

```cpp
markSourceDirty(sourceRoot);
if (!m_systemIdle) {
    m_attemptedThisIdle.remove(sourceRoot);
}
```

空闲期内的新变化继续保留为脏代次，但该素材源要等下一次新的空闲会话再处理；一次空闲会话最多全盘扫描一次。

### 2. 媒体任务增量化

修改前：

```text
素材源完成重扫 -> 按素材总数重新创建元数据与缩略图任务
```

修改后：

```text
先查询 media_metadata / thumbnail 缺失数量
仅为新增、变化或结果缺失的可读素材创建任务
结果完整时不产生空任务、不重复执行 ffprobe/缩略图
```

### 3. 应用依赖初始化顺序

修改前：

```text
AppContext 成员声明中 ScanEngine 位于 ThumbnailEngine 之前
构造初始化列表的视觉顺序无法改变 C++ 实际声明顺序
```

修改后：

```text
按真实依赖顺序声明并初始化：数据库 -> 适配器 -> 媒体/缩略图 -> 扫描 -> 业务服务 -> ViewModel
ScanEngine 构造时 ThumbnailEngine 已确定完成初始化
```

### 4. 搜索索引保留已有解析内容

修改前：

```text
ExifTool 元数据同步刷新 FTS 行 -> 可能用空值覆盖既有 AI 摘要、关键词、字幕/OCR
```

修改后：

```text
刷新前重新合并 video_analysis_result 与 video_frame_analysis
文件元数据更新不再损失已有智能解析结果
```

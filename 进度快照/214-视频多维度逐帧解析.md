## 具体修改的代码前后对比

### 多维度解析请求方式

修改前：视频多维度解析把全部帧描述拼成一个基础上下文，再一次性请求模型。

```cpp
const auto baseContext = buildDimensionBaseContext(asset, summary, frames, existingDimensions);
analyses = m_visionApiClient->analyzeDimensions(asset.fileName,
                                                baseContext,
                                                pendingDimensions,
                                                config.baseUrl,
                                                config.apiKey,
                                                config.model,
                                                config.timeoutSec,
                                                &analysisError,
                                                &httpStatusCode);
```

修改后：视频逐帧请求单帧维度结果，保存明细后再汇总。

```cpp
const auto frameAnalyses = m_visionApiClient->analyzeFrameDimensions(frame.imagePath,
                                                                     asset.fileName,
                                                                     buildFrameDimensionContext(frame),
                                                                     framePendingDimensions,
                                                                     config.baseUrl,
                                                                     config.apiKey,
                                                                     config.model,
                                                                     config.timeoutSec,
                                                                     &frameError,
                                                                     &frameHttpStatusCode);

persistDimensionFrameAnalyses(db, normalizedKey, frame, *frameAnalyses, config.model, &errorMessage);

const auto summaryContext = buildDimensionFrameSummaryContext(asset, summary, frameDimensionAnalyses);
analyses = m_visionApiClient->analyzeDimensions(asset.fileName,
                                                summaryContext,
                                                pendingDimensions,
                                                config.baseUrl,
                                                config.apiKey,
                                                config.model,
                                                config.timeoutSec,
                                                &analysisError,
                                                &httpStatusCode);
```

### 帧级明细表

修改前：只有最终维度汇总表 `material_dimension_analysis`。

修改后：新增 `material_dimension_frame_analysis`。

```sql
CREATE TABLE IF NOT EXISTS material_dimension_frame_analysis (
    video_key TEXT NOT NULL,
    dimension_key TEXT NOT NULL,
    dimension_name TEXT NOT NULL,
    frame_number INTEGER NOT NULL DEFAULT 0,
    timestamp_ms INTEGER NOT NULL DEFAULT 0,
    image_path TEXT NOT NULL DEFAULT '',
    detail TEXT NOT NULL DEFAULT '',
    error_message TEXT NOT NULL DEFAULT '',
    analysis_state INTEGER NOT NULL DEFAULT 0,
    model_name TEXT NOT NULL DEFAULT '',
    prompt_version TEXT NOT NULL DEFAULT '',
    analyzed_at TEXT NOT NULL DEFAULT '',
    UNIQUE(video_key, dimension_key, frame_number)
);
```

### 续跑规则

修改前：最终维度不存在时，会重新组织一次完整多维度请求。

修改后：

- `loadCompletedDimensionFrames(...)` 先读取已成功帧维度。
- 对每一帧只请求缺失维度。
- 成功写入 `analysis_state = Success`，失败写入 `analysis_state = Failed` 和 `error_message`。
- 下次全局多维度解析跳过成功明细，继续补齐未完成部分。

# 208-视觉请求文件名与多维度接口

- 时间：2026-07-05
- 当前模块：视觉接口请求增强

## 已完成内容

- 图片解析 `VisionApiClient::analyzeImage` 已新增 `fileName` 参数。
- 视频帧解析 `VisionApiClient::analyzeFrame` 已新增 `sourceFileName` 参数。
- `VideoAnalysisService` 调用图片解析和视频逐帧解析时，已传入素材源文件名。
- 图片/帧视觉提示中会明确包含文件名，用于辅助模型识别品牌、日期、地点、项目名、版本等线索。
- 已新增 `VisionApiClient::analyzeDimensions`，支持基于已有解析上下文按维度返回补充分析。
- 多维度接口要求模型返回：

```json
{"dimensions":[{"name":"","detail":""}]}
```

- 已补充单测验证：
  - 图片解析请求体包含文件名。
  - 多维度解析请求体包含文件名、维度名和基础解析上下文。
  - 多维度 JSON 返回可解析为 `MaterialDimensionAnalysis`。

## 具体修改的代码前后对比

详细阶段对比将在 `backup/v0.1.160/多维度解析功能-代码对比.md` 中汇总。

### 图片解析签名

修改前：

```cpp
std::optional<VisionVideoSummary> analyzeImage(const QString &imagePath, ...);
```

修改后：

```cpp
std::optional<VisionVideoSummary> analyzeImage(const QString &imagePath,
                                               const QString &fileName,
                                               ...);
```

### 图片提示词

修改前：

```text
请分析这张图片素材，只返回 JSON...
```

修改后：

```text
请结合文件名《...》分析这张图片素材，只返回 JSON...
```

### 视频帧提示词

修改前：

```text
请分析这张视频帧图片，只返回 JSON...
```

修改后：

```text
这是素材文件《...》的抽帧图片。请结合文件名线索分析这张视频帧...
```

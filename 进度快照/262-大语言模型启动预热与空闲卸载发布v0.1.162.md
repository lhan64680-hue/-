# 262 - 大语言模型启动预热与空闲卸载发布 v0.1.162

## 任务目标

1. 软件正常启动后自动加载内置 GPU 大语言模型，减少用户首次搜索等待。
2. 软件连续无操作 1 小时后自动卸载模型并释放显存。
3. 用户恢复操作后自动重新预热模型。
4. 设置页增加“大语言模型自动卸载时间”，支持手动输入和常用时间下拉选择。
5. 完成生命周期测试、全量回归、真实启动探针和新版安装包。

## 当前模块

全部开发、测试和发布工作已经完成，当前版本为 `v0.1.162`。

## 已完成内容

### 1. 启动自动预热

- 主窗口 QML 加载完成后启动模型生命周期控制器，延迟 250 毫秒开始 GPU 预热。
- 预热只在正常交互式主程序中执行；更新器会话和分析探针不会加载文本模型。
- 设置中关闭内置文本助手时，启动不加载模型并立即停止已有运行时。
- 搜索流程仍保留按需启动兜底，模型预热尚未完成时不会破坏首轮本地结果。

### 2. 应用级无操作检测

- 新增 `ApplicationIdleMonitor`，通过应用事件过滤器监听键盘、鼠标、滚轮、触摸、数位板和原生手势。
- 快捷搜索全局热键和托盘“显示主窗口”信号也被计为用户操作。
- 每次操作都会重置计时；达到超时时间后发送空闲信号。
- 空闲状态下收到任意用户操作会发送恢复信号。

### 3. 模型卸载与恢复

- 新增 `SearchAssistantLifecycleController`，统一协调设置、空闲监控和模型运行时。
- 默认连续无操作 60 分钟后停止 `llama-server`，完整释放模型和 GPU 显存。
- llama.cpp 原固定 `--sleep-idle-seconds 120` 已改为 `-1`，避免运行时自己的 2 分钟策略与软件设置冲突。
- 用户恢复操作后立即重新预热；模型运行时仍会重新探测 Vulkan GPU，禁止 CPU 回退。
- 模型运行时 `stop()` 会发出状态更新，设置页可及时刷新显示。

### 4. 自动卸载时间设置

- 新增持久化键 `materialCenter/searchAssistantAutoUnloadMinutes`。
- 默认值：60 分钟。
- 有效范围：5–1440 分钟；超出范围自动归一化。
- 手动输入：可编辑数字输入框。
- 常用下拉：15 分钟、30 分钟、1 小时（默认）、2 小时、4 小时、8 小时。
- 保存设置后立即重置空闲计时并应用新的生命周期配置。
- 状态文字显示当前 GPU、自动卸载时长、启动中、失败或空闲卸载状态。

## 关键代码前后对比

### 模型启动方式

修改前：只有搜索需要模型时才启动。

```cpp
if (!m_localSearchAssistantRuntime->isReady()) {
    m_localSearchAssistantRuntime->start();
}
```

修改后：主界面就绪后自动启动交互服务并预热，搜索路径仍保留兜底。

```cpp
m_context->startInteractiveServices();
```

### 空闲策略

修改前：llama.cpp 固定 120 秒睡眠，不理解软件内的用户活动。

```cpp
QStringLiteral("--sleep-idle-seconds"), QStringLiteral("120")
```

修改后：禁用运行时内置睡眠，由应用级控制器按设置停止进程。

```cpp
QStringLiteral("--sleep-idle-seconds"), QStringLiteral("-1")
```

```cpp
void SearchAssistantLifecycleController::unloadForIdle()
{
    m_runtime->stop();
    m_idleUnloaded = true;
}
```

### 设置持久化

```cpp
int AppSettings::searchAssistantAutoUnloadMinutes() const
{
    return qBound(5, storedMinutes, 24 * 60);
}
```

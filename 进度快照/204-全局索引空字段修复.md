## 已完成内容

- 已读取上一份最新快照：`进度快照/203-旧素材源全文件补扫.md`。
- 已创建阶段备份：`backup/v0.1.157/`。
- 已补充本模块代码对比文档：`backup/v0.1.157/全局索引空字段修复-代码对比.md`。
- 已确认截图中的“同步全局索引 002 25% · 失败”不是旧素材源补扫失败；项目库内补扫后的 `asset_file` 已包含视频、图片、文档等全文件记录。

- 已在 `MaterialCatalogSyncService` 中新增空字符串规范化，写入全局索引和 FTS 搜索表时统一把 null `QString` 转为 `""`。
- 已新增 `MaterialCatalogSyncServiceTest`，覆盖没有媒体元数据、没有文本解析结果的新素材同步场景，确认 `technical_summary` 和 `source_text` 不再写入 SQL `NULL`。
- 已重新构建 Release + FFmpeg 安装包。
- 已清理构建后的空临时目录 `output/_installer_staging`。

### 空字符串规范化

修改前：

```cpp
upsert.addBindValue(asset.technicalSummary);
upsert.addBindValue(changed ? QString() : existing.sourceText);
upsert.addBindValue(changed ? QString() : existing.errorMessage);
```

修改后：

```cpp
asset.technicalSummary = emptyIfNull(asset.technicalSummary);
asset.sourceText = changed ? QStringLiteral("") : emptyIfNull(existing.sourceText);
const auto errorMessageText = changed ? QStringLiteral("") : emptyIfNull(existing.errorMessage);

upsert.addBindValue(asset.technicalSummary);
upsert.addBindValue(asset.sourceText);
upsert.addBindValue(errorMessageText);
```

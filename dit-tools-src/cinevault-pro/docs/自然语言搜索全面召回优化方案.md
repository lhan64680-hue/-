# 自然语言搜索全面召回优化方案

## 任务目标

让用户用一句自然语言同时检索文件夹、素材、视觉帧、摘要、关键词、结构化实体、OCR、日期和素材类型。数据库中已经存在的有效文本事实不能因为新版结构化字段缺失而被整体屏蔽，同时需要避免把不同帧或不同对象上的属性错误拼接成一次命中。

## 已确认问题

1. `蓝色的牛仔裤` 已经召回了大量候选，最终结果是在严格实体验证阶段被清空，并非数据库没有数据。
2. 本地解析会把 `牛仔裤` 同时拆成实体 `牛仔裤` 和材质 `牛仔`；当旧结构化实体只有标签和颜色、没有单独的材质数组时会产生不必要的否定。
3. 帧结果已有“同一帧文本证据”降级路径，素材和文件夹结果却要求 `facts_complete=1`，导致摘要、关键词、对象、帧描述明确命中时仍被排除。
4. 词法候选采用宽召回，但排序主要使用单个最佳词，缺少对“查询词覆盖率”的奖励，容易让只命中颜色的结果与同时命中颜色、物品的结果接近。

## 优化后的检索链路

1. 查询理解：解析日期、类型、目标域、OCR、实体、颜色、材质和属性；消除实体名称内部的重叠属性，例如 `牛仔裤` 中的 `牛仔`。
2. 多路召回：继续合并数据库 LIKE/FTS、路径和本地语义向量候选，保证旧库和语义索引暂不可用时仍可搜索。
3. 证据验证：
   - 第一优先级：结构化实体在同一对象上满足全部属性。
   - 第二优先级：同一帧的 caption、tags、objects、actions、setting、entities、OCR 文本覆盖全部实体约束词。
   - 不允许把仅出现在不同无关帧上的颜色和物品拼成“同一帧命中”。
4. 排序解释：完整结构化事实获得最高置信度；同帧文本降级命中适度降权并显示原因；根据查询词覆盖率提升完整命中，压低单词噪音。
5. UI 反馈：保留查询理解、语义索引和模型辅助状态；仅把真正缺少足够证据的候选计入“未纳入结果”；搜索框提供一键清空并恢复输入焦点。

## 模块与文件

- `src/core/search/NaturalLanguageQueryParser.cpp`：本地查询消歧。
- `src/core/search/SearchQueryUnderstanding.cpp`：模型理解结果合并后的约束归一化。
- `src/core/search/SearchEngine.cpp`：词项覆盖率评分与候选排序。
- `src/application/MaterialCenterQueryService.cpp`：素材/文件夹的分批同帧证据降级验证。
- `src/ui/qml/workspaces/MaterialCenterWorkspace.qml`：主体搜索框清空按钮。
- `tests/unit/*Search*Test.cpp`、`MaterialCenterQueryServiceTest.cpp`、`MaterialCenterUiContractTest.cpp`：解析、召回、过滤和 UI 合约回归。

## 验收标准

- `蓝色的牛仔裤` 能返回包含对应摘要、关键词或同帧视觉事实的素材。
- `包含蓝色牛仔裤的帧` 仍只返回逐帧结果。
- 颜色和物品仅分散在不同无关帧时，不标记为同一帧文本证据命中。
- 结构化实体完整且匹配时仍使用最高置信验证，不被降级逻辑替代。
- 点击“清空输入”后查询和结果状态同步清空，输入框重新获得焦点。
- 相关单元测试、全量测试和 Windows Release 构建全部通过。

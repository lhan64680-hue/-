# 视觉接口 502 诊断与客户端容错修复

## 现象

素材管理中心自然语言搜索能够完成本地解析和本地语义检索，但视觉语言模型辅助阶段显示：

```text
模型理解失败，已保留本地搜索：视觉接口返回异常状态码：502
```

业务日志位于当前项目的 `logs/app.log`。Qt/QML 运行日志位于 `%APPDATA%/DIT Tools/影资管家/logs/qml-startup.log`；安装目录的 `data` 仅保存数据库、模型和语义索引，不是业务日志目录。

## 根因证据

- 2026-07-14 20:59:19 至 21:00:38，项目日志连续记录 13 次空响应体 HTTP 502。
- TCP 端口可以建立连接，但绕过 Windows 系统代理后，远端直接关闭连接且不返回 HTTP 数据。
- Windows 系统代理为 `127.0.0.1:7890`；经代理访问时，上游空连接被转换为 HTTP 502。
- `/v1/models` 健康请求同样失败。
- 仅 95 字节的纯文本 `/v1/chat/completions` 请求也在约 0.34 秒返回 502。

因此可以排除图片体积、图片编码、JSON Schema、查询文本、客户端超时和模型推理耗时。真实故障点是远端视觉模型网关无法连接后端模型服务；系统代理只负责把上游异常表现为标准 502。

## 客户端修复

`VisionApiClient` 现在：

- 将 500、502、503、504 映射为明确的服务端故障文案。
- 对上述瞬时 HTTP 故障以及非超时网络中断最多重试一次。
- 两次请求间等待 350ms，第二次仍失败立即停止，避免无限重试或持续压垮后端。
- 日志记录状态码、尝试次数、单次耗时、端点、响应体摘要和错误摘要。
- 首次失败、第二次恢复时写入恢复日志。

关键变化：

```text
修改前：单次请求失败 -> 原样显示“异常状态码：502”
修改后：首次瞬时失败 -> 350ms 退避 -> 重试一次
        重试成功 -> 正常继续
        重试失败 -> 显示“视觉模型网关无法连接后端（502），已重试 1 次仍失败”
```

## 后端恢复状态与运维建议

12345 后端已于 2026-07-14 重启并通过文本、结构化输出及真实图片请求验证。客户端能够处理瞬时网关故障，但不能恢复持续离线的远端模型进程；若后续健康请求再次返回 502，需要在视觉服务主机上检查 12345 端口后的模型进程、容器或反向代理上游，或者在应用设置中更换为可用的 OpenAI-compatible Base URL。
